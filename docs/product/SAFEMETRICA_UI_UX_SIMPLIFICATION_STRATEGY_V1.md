# SafeMetrica UI/UX Simplification Strategy v1

기준일: 2026-06-16  
작업명: SafeMetrica 역할별 홈화면 단순화 전략 v1

## 1. 목적

SafeMetrica는 기능을 많이 보여주는 앱이 아니라, 각 역할이 오늘 해야 할 일을 바로 이해하고 실행하는 안전운영 기록 SaaS로 정리한다.

앞으로 UI/UX의 기준은 아래 문장으로 고정한다.

```text
더 많이 보여주는 앱이 아니라,
필요한 순간에 필요한 것만 보이는 앱.
```

## 2. 현재 문제

현재 SafeMetrica는 기능이 빠르게 장착되면서 홈화면과 진입점이 많아졌다.

문제는 아래와 같다.

```text
기능이 많아 보이지만 처음 보는 사람은 복잡하게 느낄 수 있음
사용하지 않는 메뉴가 보이면 미완성 앱처럼 보일 수 있음
근로자, 관리자, 대표, Owner 역할이 한 화면에서 섞여 보일 위험 있음
심사위원이나 고객이 10초 안에 핵심을 이해하기 어려움
```

따라서 1라운드 멘토링과 고객 시연 전에는 기능 추가보다 “보이는 구조”를 먼저 정리한다.

## 3. 제품 인식 전환

기존 인식:

```text
기능이 많은 산업안전 앱
```

전환할 인식:

```text
오늘 현장이 무엇을 확인해야 하는지 보여주는 안전운영 OS
```

SafeMetrica는 산업안전 문서를 만드는 앱이 아니다.

위험성평가 이후의 공유, 확인, 제보, 조치, 월간보고, Export를 연결하는 안전운영 기록 SaaS다.

## 4. 역할별 홈화면 원칙

### 4.1 근로자 화면

근로자 화면은 가장 단순해야 한다.

보여줄 항목:

```text
오늘 확인할 위험요인
확인했습니다
위험제보
아차사고
개선제안
제출 완료 화면
```

숨길 항목:

```text
PTW
EBM
전체 위험성평가표
관리자 메뉴
Owner 메뉴
개발 중인 기능명
DB 또는 Export 관련 표현
```

근로자 화면의 핵심은 “읽고, 확인하고, 제보한다”이다.

### 4.2 현장관리자 화면

현장관리자는 오늘 처리해야 할 업무 중심으로 본다.

보여줄 항목:

```text
오늘 TBM 작성
음성 TBM
근로자 제보 확인
공유확인 현황
조치 필요 항목
월별 보관함
```

숨길 항목:

```text
Owner 기능
내부 Export Center
개발 중인 실험 기능
사용 빈도 낮은 메뉴
법적 판단처럼 보이는 문구
```

현장관리자 화면의 핵심은 “기록하고, 확인하고, 조치한다”이다.

### 4.3 대표 화면

대표 화면은 최소한의 신호만 보여준다.

보여줄 항목:

```text
오늘 위험 신호
처리 지연 항목
이번 달 운영기록
월간보고서 보기
```

숨길 항목:

```text
세부 입력 기능
근로자용 제출 메뉴
Owner 관리 기능
기술적 상태값
raw data
```

대표 화면의 핵심은 “위험 신호를 확인하고 운영상태를 본다”이다.

### 4.4 Owner 화면

Owner 화면은 내부 운영자용이므로 기능이 많아도 된다.

보여줄 항목:

```text
고객 세팅
source 접수
후보 검토
공유항목 생성
customer check
Version Lock
Export Center
월별 보관자료 준비
```

주의:

```text
Owner 화면은 고객과 근로자에게 노출하지 않는다.
Owner Token, API 링크, 내부 UUID, raw_payload, 환경변수명과 값은 표시하지 않는다.
```

## 5. 홈화면 숨김 기준

아래 기준에 해당하면 기본 홈화면에서 숨긴다.

```text
현재 고객이 실제로 쓰지 않는 기능
설명 없이는 이해하기 어려운 기능
개발 중인 기능
법적 책임 오해가 생길 수 있는 기능
Owner 내부 운영 기능
Export/API/DB 성격의 기능
```

숨긴 기능은 삭제하지 않는다.

```text
더보기
운영도구
내부관리
실험실
Owner Console
```

같은 하위 영역으로 이동한다.

## 6. 홈화면 노출 기준

아래 기준에 해당하면 홈화면에 남긴다.

```text
현장에서 매일 쓰는 기능
역할별로 바로 실행해야 하는 기능
고객 시연에서 10초 안에 이해되는 기능
Risk Share Pack 상품 흐름에 직접 연결되는 기능
월간보고와 Export로 이어지는 기능
```

## 7. 디자인 톤

SafeMetrica의 UI는 아래 방향으로 정리한다.

```text
딥네이비 기반
화이트 또는 밝은 카드
시안 포인트
초록/노랑/빨강 상태 신호
큰 버튼
짧은 문장
역할별 카드형 구조
```

피해야 할 것:

```text
너무 많은 메뉴
너무 어두운 화면만 연속 사용
개발자용 용어
DB 테이블명 노출
내부 API 경로 노출
법적 완료처럼 보이는 문장
```

## 8. 상태 색상 기준

```text
초록: 정상 / 완료 / 확인됨
노랑: 확인 필요 / 대기 / 보완 필요
빨강: 조치 필요 / 위험 신호 / 실패
파랑: 정보 / 공유 / 기록
회색: 숨김 / 비활성 / 내부용
```

## 9. 시연용 화면 구성 기준

멘토링과 1라운드 평가 자료에서는 아래 순서로 보여준다.

```text
1. 관리자 홈
2. TBM / 음성 TBM
3. 근로자 QR 현장참여
4. 위험제보 / 아차사고 / 개선제안
5. Risk Share Pack 공유항목
6. Version Lock
7. 근로자 위험요약
8. 월별 보관함 / Export
```

시연 중 Supabase, raw_payload, Owner Token, API Key, 내부 UUID는 보여주지 않는다.

## 10. 금지 표현

아래 표현은 UI와 발표자료에서 사용하지 않는다.

```text
법적 완료
면책 보장
과태료 방지
무재해 보장
AI 확정 판단
위험성평가 대행 완료
안전관리대행 완료
KOSHA 인정 보장
```

사용 가능한 표현:

```text
운영기록
확인자료
공유 기준
관리자 검토
고객 확인
후보 제안
월별 정리
증빙 관리
현장 운영 보조
```

## 12. 결론

SafeMetrica는 기능을 많이 보여주는 앱에서 벗어나야 한다.

앞으로의 화면 구조는 역할별로 가장 중요한 행동만 먼저 보여주고, 나머지는 내부 운영도구 또는 더보기 영역으로 숨긴다.

좋은 첫인상은 기능 수가 아니라, 사용자가 지금 무엇을 해야 하는지 바로 이해하는 화면에서 나온다.
