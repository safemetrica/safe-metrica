# SafeMetrica Risk Share Pack Commercial Launch Checklist v1

- 문서 성격: Risk Share Pack 상용 전 운영·영업·온보딩 체크리스트
- 적용 범위: 위험성평가 공유팩, 고객 온보딩, 링크/QR 배포, 월간 운영요약, 고객 전달자료, CSV Export
- 기준 상태: Risk Share Pack 관리자 홈, 공유팩 월간요약, 내부 운영자 Export, 링크/QR 운영 패널 반영 이후
- 목적: 다음 주 영업 및 초기 유료 운영 시 말과 운영 기준이 흔들리지 않도록 상용 전 확인 항목을 잠근다.

---

## 1. 상품 정의

Risk Share Pack은 위험성평가 결과가 현장 근로자에게 공유되고, 근로자 확인·의견·근로자대표 참여확인·관리자 검토·월간요약·고객 전달자료로 이어지는 운영기록 패키지다.

Risk Share Pack은 위험성평가를 대신 작성하는 서비스가 아니다.

핵심 제공 흐름:

1. 위험성평가 결과 공유
2. 근로자 공유확인
3. 위험제보 / 아차사고 / 개선제안 접수
4. 근로자대표 참여확인
5. 관리자 검토
6. 월간 운영요약
7. 내부 운영자 확인 후 고객 전달자료 정리

---

## 2. 가격 운영 후보

초기 가격 후보:

- 월 과금: 1인당 월 2,000원
- 초기 세팅비: 별도
- 월간 운영요약: 기본 제공
- CSV Export: 요청 시 / 사고 시 / 계약 종료 시 / 연말 재검토 시 제공
- QR 포스터 제작·현장 배포 지원: 별도 협의 가능

주의:

- 가격은 최종 확정 전 고객 규모, 현장 수, 운영지원 범위, 월간 리포트 제공 수준에 따라 조정한다.
- 위험성평가 대행, 안전관리대행, 법적 면책 보장으로 설명하지 않는다.

---

## 3. 신규 고객 온보딩 체크리스트

### 3.1 계약 전 확인

- 고객사명
- 사업장명 / 현장명
- 업종
- 근로자 수
- 현장관리자 이름
- 대표 또는 담당 임원
- 근로자대표 여부
- 기존 위험성평가표 보유 여부
- 위험성평가 공유 방식
- 현장 QR 게시 가능 여부
- 월간보고서 수신 담당자
- CSV Export 요청 가능 담당자

### 3.2 시스템 세팅

- Companies SSOT 등록
- company code 확정
- Risk Share Pack 사용 여부 표시
- submit-only 고객 여부 확인
- 현장참여 링크 확인
- 근로자대표 참여확인 관리 화면 확인
- 공유팩 월간요약 route 확인
- 내부 운영자 Export 패널 확인
- 고객별 링크 목록 작성

### 3.3 고객별 필수 링크

고객별로 최소 아래 링크를 정리한다.

- 관리자 홈: `/manager/risk-share`
- 근로자 공유확인·현장 의견: `/field/participation?company={companyCode}`
- 근로자대표 참여확인 관리: `/manager/representative-confirmations`
- 공유팩 월간 운영요약: `/monthly-report/risk-share`
- 내부 운영자 Export: `/manager/risk-share` 내 Export 패널

주의:

- 근로자 현장참여 링크에는 반드시 `company={companyCode}`가 포함되어야 한다.
- company query가 빠진 QR 또는 링크는 배포하지 않는다.
- 실제 token, API Key, 내부 원장 URL, Supabase URL, Notion DB URL은 고객용 안내문에 노출하지 않는다.

## 4. 링크 / QR 배포 전 확인

QR 배포 전 확인:

- 고객사명이 올바르게 표시되는가
- 회사코드가 올바르게 연결되는가
- `/field/participation?company={companyCode}` 형식인가
- 공유확인 4단계가 정상 표시되는가
- 의견 없음 제출과 위험제보 입력 흐름이 분리되는가
- 제출 후 완료 화면이 정상 표시되는가
- 관리자 홈 summary에 제출 건수가 반영되는가
- 잘못된 고객사로 이동하지 않는가

QR 안내문 포함 문구 후보:

- “위험성평가 공유확인 및 현장 의견 제출”
- “위험요인 확인 후 의견이 있으면 제출해 주세요.”
- “이 기록은 현장 운영기록으로 관리됩니다.”

QR 안내문 금지 문구:

- 법적 의무 완료
- 과태료 방지
- 면책 보장
- 무재해 보장
- 위험성평가 대행
- 안전관리대행
- QR 제출만으로 책임 완료

---

## 5. 월간 운영요약 제공 기준

매월 기본 제공:

- 근로자 공유확인 건수
- 위험제보 / 아차사고 / 개선제안 건수
- 관리자 검토 필요 건수
- 근로자대표 참여확인 건수
- 보완 의견 있음 건수
- 고객 전달자료 준비 상태

제공 방식:

- 공유팩 전용 월간요약 화면 확인
- 필요 시 인쇄 / PDF 저장
- 고객에게 월간 요약본 전달
- CSV 원본은 기본 자동 전달하지 않는다.

월간 요약 표현 기준:

- 운영기록 확인
- 공유·참여 기록
- 관리자 검토 흐름
- 다음 달 확인 필요 항목
- 고객 전달자료

금지:

- 법적 완료 리포트
- 면책자료
- 처벌 방지 자료
- 조치완료 확정 자료

---

## 6. CSV Export 제공 기준

CSV Export는 내부 운영자가 확인 후 제공한다.

기본 원칙:

- 고객이 직접 무제한 다운로드하지 않는다.
- Owner 또는 내부 운영자가 기간과 dataset을 확인 후 다운로드한다.
- 고객에게 전달해도 되는 범위인지 검토한다.
- 민감 의견, 개인정보, 내부 URL이 포함되지 않았는지 확인한다.

제공 시점:

1. 고객 요청 시
2. 사고·아차사고·분쟁·점검 대응 자료가 필요할 때
3. 대표 또는 담당자가 월간회의 자료로 요청할 때
4. 계약 종료 시 최종 운영기록 묶음으로 전달할 때
5. 연말 또는 다음 정기 위험성평가 재검토 시기

제공 dataset:

- 근로자 공유확인 CSV
- 위험제보·아차사고·개선제안 CSV
- 근로자대표 참여확인 CSV
- 증빙목록 CSV 후보

---

## 7. 사고 또는 점검 대응 시 운영기준

사고 또는 점검 이슈가 있을 때는 전체 데이터를 무작정 전달하지 않는다.

우선 범위:

- 사고일 기준 전후 30일
- 또는 사고 발생 월 전체
- 또는 해당 위험요인 관련 제출 기록

확인 항목:

- 공유확인 기록
- 위험제보 / 아차사고 / 개선제안
- 근로자대표 참여확인
- 관리자 검토 메모
- 관련 증빙 목록
- 월간 운영요약

표현 기준:

- 사후 설명자료로 활용 가능한 운영기록
- 공유·참여·검토 과정 기록
- 고객 내부 검토용 자료

금지:

- 면책자료
- 책임 회피 자료
- 처벌 방지 자료
- 법적 판단 완료 자료

---

## 8. 계약 종료 시 Export Pack

계약 종료 시 고객이 자기 운영기록을 확인할 수 있도록 최종 Export Pack을 제공할 수 있다.

구성 후보:

- 전체 근로자 공유확인 CSV
- 전체 위험제보·아차사고·개선제안 CSV
- 전체 근로자대표 참여확인 CSV
- 증빙목록 CSV
- 월간 운영요약 PDF 또는 문서

제공 전 확인:

- 기간
- 고객사명
- company code
- 민감정보 포함 여부
- 내부 URL 포함 여부
- 중복 파일 여부
- 고객 전달 가능 범위

---

영업 전 확인:

- 현장참여 링크에 company query 포함

- QR 배포 문구 확인 완료
- 금지 표현 미사용 확인

## 10. 운영 판단

Risk Share Pack은 고객 확보를 위한 진입상품이다.

초기 목표는 고객이 위험성평가 이후 공유·참여·검토·월간요약 흐름을 경험하게 하는 것이다.

이후 고객이 TBM, PTW, Evidence Book, 전체 월간보고서, 대표 대시보드가 필요하다고 느끼면 Full SafeMetrica App으로 확장한다.

상용화 초기에는 빠른 확장보다 고객별 링크 정합성, 데이터 노출 방지, 월간 전달자료 품질을 우선한다.
