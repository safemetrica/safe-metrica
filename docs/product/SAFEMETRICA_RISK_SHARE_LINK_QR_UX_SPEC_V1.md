# SafeMetrica Risk Share Link QR UX Spec v1

기준일: 2026-06-09
문서 위치: `docs/product/SAFEMETRICA_RISK_SHARE_LINK_QR_UX_SPEC_V1.md`
작업명: `docs: add risk share link qr ux spec v1`

## 1. 목적

이 문서는 `SafeMetrica Risk Share Link` 상품의 QR/링크 기반 사용자 경험 기준을 정리하기 위한 UX Spec이다.

이전 문서 `SafeMetrica Risk Share Link Product Spec v1`은 상품의 범위, 대상 고객, 가격 후보, 제공 범위, 제외 범위, 법적 표현 기준을 정리했다.

본 문서는 그 상품을 실제 현장에서 어떻게 쓰게 할지 정리한다.

핵심 목적은 다음과 같다.

```text
근로자는 QR로 쉽게 확인하고,
관리자는 공유확인과 위험제보를 분리해서 보고,
대표는 월간 요약과 고객용 Export 후보로 확인한다.
```

Risk Share Link는 전체 SafeMetrica 앱을 바로 도입하기 어려운 사업장에도 위험성평가 공유·근로자 참여·의견제출 기록을 먼저 운영할 수 있게 하는 진입형 QR 상품이다.

## 2. UX 기본 원칙

Risk Share Link QR UX는 다음 원칙을 따른다.

```text
근로자 입력은 짧게
관리자 확인은 명확하게
대표 보고는 숫자 중심
법적 표현은 보수적으로
AI는 후보 제안자로만 표시
```

모바일 우선 원칙:

```text
1. QR 접속 후 10초 안에 무엇을 해야 하는지 보여준다.
2. 근로자 화면에는 긴 설명을 넣지 않는다.
3. 체크 항목은 2~3개로 제한한다.
4. 위험요인은 태그와 짧은 문장 중심으로 보여준다.
5. 의견 없음 제출과 위험제보 제출을 분리한다.
6. 제출 완료 후 근로자가 다시 수정해야 하는 부담을 줄인다.
```

## 3. 사용자 역할

Risk Share Link v1의 역할은 3개로 본다.

| 역할        | 주요 목적                         |
| --------- | ----------------------------- |
| 근로자       | 위험성평가 공유 확인, 안전조치 주지 확인, 의견제출 |
| 현장관리자     | 공유확인 현황 확인, 위험제보 검토, 처리상태 관리  |
| 대표/고객 담당자 | 월간 요약, 미검토 건수, 조치상태 확인        |

v1에서는 Owner Console을 외부 사용자에게 제공하지 않는다.

## 4. 전체 흐름

기본 흐름은 다음과 같다.

```text
관리자
→ 회사/현장별 QR 또는 링크 준비
→ 현장 게시 또는 근로자에게 전달

근로자
→ QR 접속
→ 오늘 위험요인 확인
→ 위험성평가 공유 확인
→ 안전조치 주지 확인
→ 의견 없음 또는 의견 제출
→ 제출 완료

관리자
→ 접수함 확인
→ 공유확인과 검토대상 분리
→ 위험제보/아차사고/개선제안 처리
→ 조치 메모 기록

대표/고객 담당자
→ 월간 요약 확인
→ 고객용 Export 후보 확인
```

## 5. QR 접근 방식

### 5-1. QR 유형

QR은 아래 3가지 유형을 후보로 둔다.

| QR 유형    | 설명           | v1 우선순위 |
| -------- | ------------ | ------- |
| 회사 공통 QR | 회사 전체 근로자용   | 높음      |
| 현장별 QR   | 사업장/현장별 구분   | 높음      |
| 작업별 QR   | 특정 작업·공정별 구분 | 후순위     |

v1에서는 회사 공통 QR 또는 현장별 QR을 우선한다.

작업별 QR은 운영 복잡도가 높으므로 후속 UX로 분리한다.

### 5-2. QR 게시 위치 후보

```text
TBM 장소
출근 동선
작업장 입구
휴게실
차고지
안전게시판
관리사무소
장비 보관구역
위험성평가표 게시 위치
```

### 5-3. QR 안내 문구

게시물 문구 후보:

```text
오늘 작업 전 위험요인을 확인해 주세요.
위험성평가 공유 확인과 현장 의견을 QR로 제출할 수 있습니다.
```

더 짧은 문구:

```text
위험요인 확인 / 의견제출 QR
```

금지 문구:

```text
이 QR로 법적 의무 완료
이 QR로 과태료 방지
이 QR로 중대재해처벌법 대응 완료
근로자 참여 의무 자동 충족
```

## 6. 근로자 화면 구조

근로자 화면은 5단계로 구성한다.

```text
1. 회사/현장 확인
2. 오늘 위험요인 확인
3. 위험성평가 공유 확인
4. 안전조치 주지 확인
5. 의견 없음 또는 의견 제출
```

### 6-1. 상단 영역

상단에는 아래 정보만 표시한다.

```text
회사명
현장명 또는 작업장명
오늘 날짜
간단 안내문
```

예시:

```text
대도환경
오늘 작업 전 위험요인을 확인해 주세요.
```

### 6-2. 위험요인 표시

위험요인은 태그와 1줄 설명으로 표시한다.

예시:

```text
[후진] [협착] [미끄럼] [찔림·베임]

오늘 작업 중 차량 후진, 적재함 주변 접근, 젖은 바닥 미끄럼에 주의가 필요합니다.
```

위험요인은 최대 5개까지만 우선 노출한다.

5개 초과 시:

```text
+3개 더보기
```

### 6-3. 확인 체크

근로자 확인 체크는 v1에서 2개를 기본으로 한다.

```text
□ 위험성평가 내용을 확인했습니다.
□ 오늘 안전조치와 주의사항을 확인했습니다.
```

현장에 따라 선택 후보:

```text
□ 보호구 착용 기준을 확인했습니다.
□ 작업 전 이상 여부를 확인했습니다.
□ 온열질환 자각증상을 확인했습니다.
```

선택 후보는 기상위험이나 작업유형에 따라 후속 단계에서 조건부 표시한다.

## 7. 의견 없음 / 의견 제출 분기

Risk Share Link의 핵심은 공유확인과 위험제보를 섞지 않는 것이다.

근로자 화면 하단에는 두 가지 버튼을 둔다.

```text
의견 없음, 확인만 제출
위험 또는 개선의견 제출
```

### 7-1. 의견 없음 제출

근로자가 특별한 의견이 없을 때:

```text
제출구분 = 공유확인
처리상태 = 접수 또는 확인완료 후보
내용 = 의견 없음
```

주의:

```text
공유확인은 조치완료 KPI에 섞지 않는다.
```

### 7-2. 위험 또는 개선의견 제출

근로자가 의견을 제출할 때 선택값은 다음으로 한다.

```text
위험제보
아차사고
개선제안
기타
```

입력 필드는 최소화한다.

```text
제목 또는 한 줄 요약
위치/구역
내용
사진첨부 선택
익명 여부 선택
```

## 8. 근로자 제출 완료 화면

제출 후에는 간단한 완료 화면을 보여준다.

### 8-1. 공유확인 완료

```text
확인 기록이 제출되었습니다.
오늘 작업 전 위험요인과 안전조치 확인 기록으로 저장됩니다.
```

### 8-2. 위험제보 완료

```text
의견이 접수되었습니다.
현장관리자가 내용을 확인하고 필요한 경우 조치사항을 기록합니다.
```

금지 표현:

```text
조치가 완료되었습니다.
법적 증빙이 완료되었습니다.
위험성평가가 완료되었습니다.
```

## 9. 중복 제출 방지 UX

근로자 제출 버튼은 중복 클릭을 방지해야 한다.

v1 UX 기준:

```text
제출 버튼 클릭 즉시 비활성화
버튼 문구를 "제출 중입니다..."로 변경
완료 전 재클릭 방지
제출 완료 후 완료 화면으로 이동
```

서버 저장 구조는 구현 Spec에서 확정한다.

단, UX 기준으로는 다음 원칙을 둔다.

```text
동일 사용자가 같은 내용으로 여러 번 제출되는 것을 줄인다.
사진 첨부가 있는 경우 중복 업로드를 방지한다.
제출 중에는 뒤로가기보다 대기 안내를 우선한다.
```

## 10. 관리자 접수함 UX

관리자 접수함은 공유확인과 검토대상을 분리한다.

### 10-1. 상단 요약

```text
오늘 공유확인: 00건
검토 필요: 00건
조치 필요: 00건
조치완료: 00건
```

### 10-2. 탭 구조

```text
공유확인
검토 필요
조치 필요
조치완료
반려
전체
```

### 10-3. 카드 표시 기준

공유확인 카드:

```text
근로자명 또는 익명
제출시간
확인 항목
의견 없음
```

위험제보 카드:

```text
제출구분
위치/구역
제목
내용 요약
사진 여부
처리상태
조치 메모
```

### 10-4. 관리자 처리 버튼

```text
검토중으로 변경
조치필요로 변경
조치완료로 변경
반려
조치 메모 작성
사진 보기
```

## 11. 대표/고객 요약 UX

Risk Share Link 단품에서는 대표 대시보드 전체를 제공하지 않는다.

다만 월간 요약 또는 간단 카드 형태로 아래를 제공할 수 있다.

```text
이번 달 공유확인 수
위험제보 수
아차사고 수
개선제안 수
미검토 건수
조치필요 건수
조치완료 건수
재발 위험 키워드 TOP 3
```

대표용 문구는 짧게 유지한다.

예시:

```text
이번 달 근로자 공유확인 128건, 위험제보 4건, 조치완료 3건입니다.
미검토 1건은 확인이 필요합니다.
```

주의:

```text
공유확인 수를 조치완료 수로 표시하지 않는다.
위험제보 조치완료율 계산 시 공유확인은 제외한다.
```

## 12. 모바일 UI 문구 기준

근로자 화면 문구는 쉽게 쓴다.

권장 표현:

```text
오늘 위험요인을 확인해 주세요.
확인 후 제출해 주세요.
위험하거나 불편한 점이 있으면 의견을 남겨 주세요.
사진은 선택입니다.
익명 제출도 가능합니다.
```

관리자 화면 문구:

```text
공유확인 기록입니다.
검토가 필요한 의견입니다.
조치 메모를 남겨 주세요.
사진을 확인해 주세요.
```

금지 또는 지양 표현:

```text
법적 의무 완료
책임 회피
처벌 방지
면책 자료
근로자 동의 확보 완료
위험성평가 인정 완료
```

## 13. 접근권한 기준

### 13-1. 근로자 QR

근로자 QR은 접근성을 우선한다.

v1 기준:

```text
로그인 없이 접근 가능 후보
회사/현장 코드 기반 접근
민감한 관리자 정보 비노출
대표/관리자 화면 접근 불가
```

### 13-2. 관리자 화면

관리자 화면은 보호한다.

```text
회사별 관리자 접근 링크 또는 로그인 후보
Owner Console 외부 공유 금지
토큰값 문서 기록 금지
실제 고객 데이터 체험판 사용 금지
```

### 13-3. 링크 유출 대응

링크 유출 가능성을 고려해 후속 단계에서 아래를 검토한다.

```text
토큰 만료
현장별 QR 재발급
접근 로그
IP 또는 기기 제한 후보
GPS 반경 인증 후보
```

v1에서는 강제 위치 인증이나 본인인증은 하지 않는다.

## 14. 사진첨부 UX

사진첨부는 선택으로 둔다.

근로자 화면 문구:

```text
사진이 있으면 첨부해 주세요. 선택사항입니다.
```

사진 후보:

```text
위험장소
미끄럼 구간
파손 부위
적재 상태
차량 주변
조치 전 상태
개선 제안 위치
```

사진첨부는 위험제보, 아차사고, 개선제안에 우선 연결한다.

공유확인만 제출하는 경우 사진첨부는 기본 노출하지 않거나 접힘 처리한다.

## 15. 기상 위험 연결

기상 위험 Level 2 이상일 때 근로자 화면에 짧은 확인 항목을 추가할 수 있다.

폭염 예시:

```text
오늘은 온열질환 주의가 필요합니다.
물, 그늘, 휴식 준비상태를 확인해 주세요.
어지러움, 두통, 근육경련이 있으면 의견을 남겨 주세요.
```

체크 후보:

```text
□ 온열질환 자각증상을 확인했습니다.
```

강풍 예시:

```text
강풍 위험이 있는 작업은 작업 전 상태 확인이 필요합니다.
고소작업, 적재물, 외부 작업구역을 확인해 주세요.
```

주의:

```text
기상 위험 카드는 작업중지 명령이 아니다.
기상 위험 문구는 작업 전 확인 안내로 표시한다.
```

## 16. 교육/TBM 인정 후보 연결

Risk Share Link는 교육 이수나 TBM 인정 여부를 자동 확정하지 않는다.

다만 공유확인 기록은 교육·TBM 운영기록 후보로 활용될 수 있다.

표시 문구 후보:

```text
교육/TBM 기록 후보
관리자 확인 필요
공유확인 기록
```

주의 문구:

```text
교육시간 인정 여부는 사업장 조건, 교육 내용, 대상자, 시간, 증빙, 관련 기준 확인 후 판단합니다.
```

금지 표현:


## 21. 최종 정리

Risk Share Link QR UX의 핵심은 “위험성평가 공유를 실제 근로자 확인과 의견제출 기록으로 연결하는 것”이다.

최종 UX 원칙은 다음과 같다.

```text
QR로 쉽게 접근
위험요인은 짧게 확인
공유확인과 위험제보는 분리
사진첨부는 선택
관리자는 검토·조치 기록
대표는 숫자로 요약 확인
Export는 고객 전달용으로 정제
```

최종 한 줄:

```text
Risk Share Link QR UX는 위험성평가를 사무실 문서에서 끝내지 않고,
근로자의 확인·의견제출·관리자 검토 기록으로 연결하는 SafeMetrica의 현장형 진입 UX다.
```
