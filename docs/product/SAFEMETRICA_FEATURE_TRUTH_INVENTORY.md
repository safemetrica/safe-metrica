# SafeMetrica Feature Truth Inventory

기준일: 2026-06-07
문서 목적: SafeMetrica 정식앱, Partner Demo, 랜딩페이지, 영업자료가 서로 다른 기능 약속을 하지 않도록 기능 상태와 외부 표현 기준을 고정한다.

---

## 1. 목적

SafeMetrica는 근로자의 확인과 제보, 현장관리자의 조치, 대표의 운영 확인을 하나의 안전운영 기록으로 연결하는 산업안전 운영기록 SaaS다.

이 문서는 다음 혼선을 방지하기 위해 작성한다.

* 정식앱에서 제공하는 기능과 Partner Demo 샘플 기능이 섞이는 문제
* 랜딩페이지나 제안서에서 아직 제공하지 않는 기능을 제공하는 것처럼 표현하는 문제
* AI가 법적 판단자 또는 조치 확정자인 것처럼 보이는 문제
* 장기 비전 기능이 현재 운영 기능처럼 오해되는 문제
* 실제 고객 데이터, API, DB, 토큰, 환경변수 값이 체험판이나 문서에 노출되는 문제

## 6. Partner Demo 노출 원칙

Partner Demo는 정식앱의 과장판이 아니다.

Partner Demo는 정식앱 핵심 흐름의 샘플 모드다.

Partner Demo는 아래 흐름만 명확히 보여준다.

1. 근로자: TBM 확인 → 위험성평가 공유확인 → 위험제보 제출
2. 현장관리자: 참여현황 확인 → 위험제보 접수 → 조치사진 샘플 저장 → 현장비서 요약
3. 대표: 운영 대시보드 → 위험성평가표 샘플 → 월간보고서 샘플

Partner Demo에서 금지되는 항목은 다음과 같다.

* 실제 고객명
* 실제 고객 사진
* 실제 고객 민감정보
* 실제 Notion DB
* Supabase 직접 연결
* API 직접 호출
* Owner Console
* 토큰
* API Key
* 환경변수 값
* 실제 고객 운영 링크
* 실제 고객 데이터 기반 월간보고서
* 실제 고객 위험성평가 원본

Partner Demo는 샘플 데이터와 `localStorage` 기반 샘플 상태만 사용한다.

## 7. 정식앱 홈 노출 원칙

정식앱 홈은 장기적으로 아래 혼합형 구조로 간다.

역할별 오늘 할 일
+
전체 기능 메뉴

운영 원칙은 다음과 같다.

* 역할형 홈을 1차 내비게이션으로 둔다.
* 기존 기능 메뉴는 2차 내비게이션으로 유지한다.
* 대표, 현장관리자, 근로자 관점의 오늘 할 일을 먼저 보여준다.
* 실제 기능이 미완성인 항목은 준비 중 또는 장기 검토로 표시한다.
* 고객사별 모듈 사용 여부에 따라 메뉴 노출을 다르게 할 수 있다.
* Risk DB 미사용 테넌트는 위험성평가 관련 기능을 제한 또는 안내 처리한다.
* Companies 또는 Tenant Config를 단일 기준으로 삼는다.

## 8. 영업자료 표현 원칙

영업자료, 랜딩페이지, 제안서, 발표자료에서는 아래 원칙을 적용한다.

* 운영중, 제한 운영, 베타 기능만 구체적으로 설명한다.
* 준비 중 기능은 “준비 중”, “고도화 예정”, “검토 중”으로 표현한다.
* 장기 검토 기능은 “로드맵”, “비전”, “확장 방향”으로만 표현한다.
* 미제공 기능은 현재 기능처럼 표현하지 않는다.
* AI는 법적 판단자나 조치 확정자가 아니라 후보 제안자다.
* 최종 판단과 책임은 관리자와 사업주에게 있다.
* SafeMetrica는 위험성평가 대행, 안전관리대행, 법률자문, 노무자문을 제공하는 것으로 표현하지 않는다.
* 상표 관련 표현은 등록 완료가 아니라 “상표 출원 진행 중”으로 표시한다.

권장 표현:

* 안전보건 관리·조치·확인 과정을 기록으로 남깁니다.
* 현장의 확인과 제보, 관리자의 조치, 대표의 운영 확인을 하나의 흐름으로 연결합니다.
* 입력된 운영기록을 월간보고서와 증빙 정리에 활용할 수 있습니다.
* AI는 누락 가능성이나 조치 후보를 제안하고, 최종 확인은 관리자가 수행합니다.

## 11. PR 체크리스트

기능 상태가 변경되거나 새 기능이 추가될 때는 아래 항목을 확인한다.

* [ ] 이 기능이 정식앱에서 실제 제공되는가?
* [ ] Partner Demo에 노출해도 되는가?
* [ ] 샘플 모드와 실제 운영 모드가 구분되는가?
* [ ] 영업자료에서 표현 가능한 수준인가?
* [ ] 금지 표현이 포함되어 있지 않은가?
* [ ] 실제 고객명, 사진, DB, API, 토큰, 환경변수 값이 노출되지 않는가?
* [ ] AI가 확정자처럼 표현되지 않는가?
* [ ] 관리자와 사업주의 최종 판단 책임이 유지되는가?
* [ ] 장기 비전 기능이 현재 기능처럼 표현되지 않는가?

## 12. 고정 결론

SafeMetrica는 기능이 많은 앱이 아니라, 안전운영 기록의 정합성을 지키는 SaaS다.

정식앱은 실제 운영기록을 남기고, Partner Demo는 그 핵심 흐름을 샘플로 체험하게 한다.

영업자료는 현재 제공 가능한 기능과 장기 비전을 구분해서 표현한다.

AI는 후보 제안자이며, 최종 판단과 책임은 관리자와 사업주에게 있다.
