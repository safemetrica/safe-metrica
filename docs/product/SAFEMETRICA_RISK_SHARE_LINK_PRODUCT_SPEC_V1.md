# SafeMetrica Risk Share Link Product Spec v1

기준일: 2026-06-09
문서 위치: `docs/product/SAFEMETRICA_RISK_SHARE_LINK_PRODUCT_SPEC_V1.md`
작업명: `docs: add risk share link product spec v1`

## 1. 목적

이 문서는 SafeMetrica 정식앱 전체 도입 전, 위험성평가 공유·근로자 참여·의견제출 증빙만 QR 또는 링크로 먼저 운영할 수 있는 단품 상품인 `SafeMetrica Risk Share Link`의 상품 기준을 정리하기 위한 문서다.

SafeMetrica 전체 기능에는 TBM, PTW, Evidence Book, 현장참여, 대표 대시보드, 월간보고서, 고객용 Export 등이 포함된다.

그러나 일부 사업장은 전체 앱 도입 전 다음 요구만 먼저 가질 수 있다.

```text
위험성평가 내용을 근로자에게 공유했는가?
근로자가 위험요인을 확인했는가?
근로자가 의견을 낼 수 있었는가?
현장 의견과 공유확인 기록이 남았는가?
관리자가 검토하고 조치 기록을 남길 수 있는가?
```

Risk Share Link는 이 흐름만 단품으로 제공하는 진입형 상품이다.

## 2. 상품 정의

상품명 후보:

```text
SafeMetrica Risk Share Link
위험성평가 공유·참여 증빙 QR팩
근로자 참여 증빙 QR팩
```

기본 정의:

```text
Risk Share Link는 근로자가 QR 또는 링크로 접속하여
위험성평가 공유 확인, 안전조치 주지 확인, 의견제출, 사진첨부를 할 수 있게 하고,
관리자가 이를 접수·검토·처리상태로 관리할 수 있게 하는
SafeMetrica의 위험성평가 공유·참여 기록 단품 상품이다.
```

## 3. 대상 고객

### 3-1. 1차 대상

Risk Share Link는 다음 사업장을 1차 대상으로 한다.

```text
50인 이상 사업장
위험성평가 공유·참여 기록이 필요한 사업장
근로자 의견청취 기록을 남기고 싶은 사업장
전체 SafeMetrica 앱 도입 전 가볍게 시작하려는 사업장
종이 서명부, 카카오톡, 엑셀, 사진첩으로 기록이 흩어진 사업장
현장관리자 수는 적고 근로자 수가 많은 사업장
```

### 3-2. 적합 업종 후보

```text
생활폐기물
건설폐기물
물류센터
제조업
도급·용역 작업장이 있는 사업장
지게차·차량·장비 이동이 많은 현장
야외작업 또는 기상 리스크가 있는 현장
```

## 4. 해결하려는 문제

기존 현장에서는 위험성평가 공유와 근로자 참여가 다음처럼 흩어지는 경우가 많다.

```text
위험성평가표는 사무실 파일에 있음
근로자는 내용을 제대로 봤는지 확인이 어려움
의견청취는 구두로 끝남
사진은 카카오톡에 흩어짐
서명부는 종이로 보관됨
대표나 관리자는 누가 확인했는지 즉시 보기 어려움
월간 보고나 고객 전달용 자료로 정리하기 어려움
```

Risk Share Link는 이 중 “공유·확인·의견제출·검토기록”만 먼저 구조화한다.

## 5. 핵심 흐름

기본 사용자 흐름은 다음과 같다.

```text
관리자
→ 위험성평가 공유 링크 또는 QR 생성
→ 현장 근로자에게 안내

근로자
→ QR 또는 링크 접속
→ 오늘 위험요인 확인
→ 위험성평가 공유 확인
→ 안전조치 주지 확인
→ 의견 없음 또는 위험제보 제출
→ 사진 첨부 가능

관리자
→ 접수함에서 공유확인과 위험제보 분리 확인
→ 위험제보·아차사고·개선제안 검토
→ 처리상태 및 조치 메모 기록

대표 또는 고객
→ 월간 요약 또는 고객용 Export로 확인
```

## 6. 제공 범위

Risk Share Link v1 제공 범위는 다음과 같다.

| 구분            | 제공 내용                      |
| ------------- | -------------------------- |
| 근로자 QR/링크     | 회사별 또는 현장별 접근 링크 제공        |
| 위험요인 확인       | 오늘 또는 해당 작업의 주요 위험요인 표시    |
| 위험성평가 공유 확인   | 근로자의 확인 기록 저장              |
| 안전조치 주지 확인    | 안전조치 확인 기록 저장              |
| 의견제출          | 위험제보, 아차사고, 개선제안, 기타 의견 제출 |
| 사진첨부          | 현장사진 또는 조치사진 후보 첨부         |
| 관리자 접수함       | 공유확인과 검토대상 분리 표시           |
| 처리상태          | 접수, 검토중, 조치필요, 조치완료, 반려    |
| 조치 메모         | 관리자 조치내용 기록                |
| 월간 요약 후보      | 공유확인 수, 검토대상 수, 조치상태 요약    |
| 고객용 Export 후보 | CSV 또는 Excel 반영 후보         |

## 7. 제외 범위

Risk Share Link v1은 SafeMetrica 전체 앱이 아니다.

v1에서 제외하는 범위는 다음과 같다.

```text
전체 TBM 운영
TBM 음성작성
PTW 고위험작업허가 전체 기능
Evidence Book 전체 기능
대표 대시보드 전체 기능
Owner Console
Supabase 원장 직접 노출
Notion 원본 DB 직접 노출
고객사 전체 앱 메뉴
법률자문
노무자문
안전관리대행
위험성평가 대행
KOSHA 인정심사 컨설팅
교육 이수 확정
법적 적합성 보장
과태료 방지 보장
중대재해 면책 보장
```

필요 시 위 기능은 SafeMetrica 정식앱 또는 별도 컨설팅 계약으로 분리한다.

## 8. 데이터 구조 기준

Risk Share Link는 현장참여 데이터 구조를 기반으로 한다.

핵심 필드:

```text
companyCode
siteCode
workerName 또는 익명 여부
제출구분
처리상태
위험요인 확인
위험성평가 공유 확인
안전조치 주지 확인
의견 제목
의견 내용
위치/구역
사진/파일
관리자 조치 메모
작성일
처리일
```

제출구분 기준:

```text
공유확인
위험제보
아차사고
개선제안
기타
```

처리상태 기준:

```text
접수
검토중
조치필요
조치완료
반려
```

핵심 원칙:

```text
공유확인과 위험제보는 같은 통계로 섞지 않는다.
공유확인은 교육·주지·참여 기록으로 본다.
위험제보, 아차사고, 개선제안은 검토대상 및 조치 KPI 후보로 본다.
```

## 9. 화면 구성 기준

### 9-1. 근로자 화면

근로자 화면은 모바일 우선으로 구성한다.

```text
1. 회사명 / 현장명
2. 오늘 확인할 위험요인
3. 위험성평가 공유 확인 체크
4. 안전조치 주지 확인 체크
5. 의견 없음 / 의견 제출 선택
6. 위험제보·아차사고·개선제안 입력
7. 사진첨부
8. 제출 완료 안내
```

근로자 화면의 원칙:

```text
입력은 짧게
버튼은 크게
문장은 쉽게
법적 표현은 최소화
관리자용 내부 용어는 숨김
```

### 9-2. 관리자 화면

관리자 화면은 접수함 중심으로 구성한다.

```text
공유확인
검토 필요
조치 필요
조치완료
반려
```

관리자는 다음을 할 수 있다.

```text
기록 확인
사진 확인
처리상태 변경
조치 메모 작성
월간보고서 반영 후보 확인
고객용 Export 후보 확인
```

### 9-3. 대표 화면 후보

Risk Share Link 단품에서는 대표 대시보드 전체를 제공하지 않는다.

다만 아래 요약 카드는 제공 후보로 둘 수 있다.

```text
이번 달 공유확인 수
위험제보 수
조치필요 수
조치완료 수
미검토 건수
재발 위험 키워드 TOP 3
```

## 11. 영업 포지션

Risk Share Link의 영업 포지션은 다음과 같다.

```text
전체 안전관리 플랫폼 도입이 부담스러운 사업장도
위험성평가 공유·근로자 참여 기록부터 가볍게 시작할 수 있습니다.
```

핵심 영업 문장:

```text
위험성평가를 작성하는 것에서 끝내지 않고,
근로자가 확인하고 의견을 남길 수 있었는지를 기록으로 남깁니다.
```

짧은 문장:

```text
위험성평가 공유와 근로자 참여를 QR로 기록합니다.
```

## 12. 법적·표현상 주의사항

Risk Share Link는 법적 면책이나 과태료 방지를 보장하지 않는다.

필수 문구:

```text
본 기능은 사업장의 위험성평가 공유, 근로자 확인, 의견제출, 관리자 검토 과정을 기록으로 남기기 위한 운영지원 기능입니다.
위험성평가의 최종 실시, 위험성 결정, 개선대책 확정, 현장 조치 실행 및 법적 책임은 사업주에게 있습니다.
```

금지 표현:

```text
위험성평가 법적 완료
근로자 참여 의무 자동 충족
과태료 방지
중대재해 면책
법적 방어 보장
교육 인정 보장
무재해 보장
KOSHA 인정 보장
```

권장 표현:

```text
공유·확인 과정을 기록으로 남김
근로자 의견제출 경로 제공
관리자 검토 기록 보관
월간 운영자료로 활용 가능
사후 설명자료 후보로 활용 가능
```

## 13. 고객용 Export 연결

Risk Share Link 데이터는 향후 고객용 Export에 연결할 수 있다.

Export 후보:

```text
공유확인 CSV
위험제보 CSV
조치상태 CSV
사진증빙 목록 CSV
월간 요약 PDF
```

고객용 Export에는 아래 정보를 포함하지 않는다.

```text
raw payload
Supabase 내부 UUID
Notion URL
Owner/Admin 링크
API Key
환경변수명과 값
토큰 또는 토큰 유사 문자열
내부 디버그 메시지
```

## 14. Supabase 전환 기준

Risk Share Link v1은 초기에는 기존 현장참여 저장 구조를 활용할 수 있다.

장기적으로는 Supabase 원장 구조에 연결한다.

권장 구조:

```text
companies
→ sites
→ risk_instances
→ field_participation_submissions
→ actions
→ evidence_items
```

단, v1 문서에서는 실제 DB 구현을 확정하지 않는다.

먼저 상품 범위와 UX 기준을 잠그고, 이후 구현 Spec에서 저장 구조를 확정한다.

## 18. 최종 정리

Risk Share Link는 SafeMetrica 전체 앱을 가볍게 쪼갠 진입형 상품이다.

전체 앱을 바로 도입하기 어려운 사업장에도 다음 가치를 먼저 제공한다.

```text
위험성평가 공유
근로자 확인
의견제출
사진첨부
관리자 검토
월간 요약
고객용 Export 후보
```

최종 한 줄:

```text
Risk Share Link는 위험성평가를 문서 보관에서 끝내지 않고,
근로자 공유·참여·의견제출 기록으로 연결하는 SafeMetrica의 진입형 QR 상품이다.
```
