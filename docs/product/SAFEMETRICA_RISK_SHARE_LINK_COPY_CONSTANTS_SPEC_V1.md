# SafeMetrica Risk Share Link Copy Constants Spec v1

기준일: 2026-06-09
문서 위치: `docs/product/SAFEMETRICA_RISK_SHARE_LINK_COPY_CONSTANTS_SPEC_V1.md`
작업명: `docs: add risk share link copy constants spec v1`

## 1. 목적

이 문서는 `SafeMetrica Risk Share Link` 관련 문구를 실제 앱 코드에 반영하기 전, 어떤 문구를 상수화할지 정리하기 위한 Spec이다.

지금까지 Risk Share Link는 아래 문서로 기준이 잠겼다.

```text
#380 Product Spec
#381 QR UX Spec
#382 Feature Truth Map
#383 Pricing v1
#384 One Page Proposal v1
#385 QR Poster Copy v1
#386 Contract / Estimate Copy v1
#387 T/A · Field Sales Script v1
```

이 문서의 목적은 위 문서들에 흩어진 문구 중 실제 앱 화면, Partner Demo, QR 화면, 관리자 접수함, Export, 포스터 출력에 사용할 문구를 코드 상수로 옮길 기준을 정하는 것이다.

## 2. 기본 원칙

Risk Share Link 문구 상수화는 다음 원칙을 따른다.

```text
문구는 한곳에서 관리한다.
화면마다 같은 의미를 다른 표현으로 쓰지 않는다.
금지 표현은 코드 레벨에서 차단한다.
근로자 화면은 짧고 쉽게 쓴다.
관리자 화면은 상태와 조치 중심으로 쓴다.
대표 화면은 숫자와 요약 중심으로 쓴다.
AI는 후보 제안자이며 최종 판단은 관리자와 사업주에게 있다.
```

## 3. 코드 위치 후보

실제 구현 시 문구 상수 파일 위치 후보는 다음과 같다.

```text
src/lib/risk-share-link/copy.ts
```

추가 분리 후보:

```text
src/lib/risk-share-link/industry-copy.ts
src/lib/risk-share-link/legal-guard-copy.ts
src/lib/risk-share-link/demo-copy.ts
src/lib/risk-share-link/export-copy.ts
```

v1에서는 우선 단일 파일로 시작할 수 있다.

```text
src/lib/risk-share-link/copy.ts
```

규모가 커지면 업종별 문구와 Demo 문구를 분리한다.

| 구분              | 상수화 필요 여부 | 설명                |
| --------------- | --------- | ----------------- |
| 근로자 QR 화면 문구    | 필요        | 실제 사용자 화면에 표시     |
| 확인 체크 문구        | 필요        | 공유확인·안전조치 확인      |
| 제출 버튼 문구        | 필요        | 의견 없음 / 위험제보 분기   |

| 법적 주의 문구        | 필요        | 금지표현 방지           |

| 계약서 문구          | 후순위       | 앱 화면에는 일부만 필요     |

## 5. Worker QR Copy

근로자 QR 화면의 기본 문구는 짧게 유지한다.

### 5-1. 화면 제목

```text
위험요인 확인 / 의견제출
```

### 5-2. 안내문

```text
오늘 작업 전 위험요인을 확인해 주세요.
위험하거나 불편한 점이 있으면 의견을 남길 수 있습니다.
```

### 5-3. 확인 체크 문구

```text
위험성평가 내용을 확인했습니다.
오늘 안전조치와 주의사항을 확인했습니다.
```

### 5-4. 선택 체크 후보

```text
보호구 착용 기준을 확인했습니다.
작업 전 이상 여부를 확인했습니다.
온열질환 자각증상을 확인했습니다.
```

선택 체크는 작업유형, 업종, 기상위험 단계에 따라 조건부 표시한다.

## 6. Worker Button Copy

근로자 화면 버튼 문구는 다음을 우선 사용한다.

```text
위험요인 확인하기
확인 후 제출하기
의견 없음, 확인만 제출
위험 또는 개선의견 제출
사진 첨부하기
제출 중입니다...
제출 완료
```

피해야 할 버튼 문구:

```text
법적 확인 완료
위험성평가 완료
교육 인정 제출
의무 이행 완료
면책자료 제출
```

## 7. Worker Completion Copy

### 7-1. 공유확인 완료

```text
확인 기록이 제출되었습니다.
오늘 작업 전 위험요인과 안전조치 확인 기록으로 저장됩니다.
```

### 7-2. 위험제보 완료

```text
의견이 접수되었습니다.
현장관리자가 내용을 확인하고 필요한 경우 조치사항을 기록합니다.
```

### 7-3. 사진첨부 완료

```text
사진이 함께 제출되었습니다.
현장 확인과 조치 검토 자료로 활용될 수 있습니다.
```

## 8. Submission Type Copy

제출구분은 아래 값을 표준으로 한다.

```text
공유확인
위험제보
아차사고
개선제안
기타
```

설명 문구:

| 제출구분 | 설명                     |
| ---- | ---------------------- |
| 공유확인 | 위험성평가와 안전조치 내용을 확인한 기록 |
| 위험제보 | 위험요인 또는 불안전 상태 제보      |
| 아차사고 | 사고로 이어질 뻔한 상황          |
| 개선제안 | 작업환경·동선·보호구·설비 개선 의견   |
| 기타   | 분류가 어려운 현장 의견          |

주의:

```text
공유확인은 조치완료 KPI에 섞지 않는다.
```

## 9. Manager Inbox Copy

관리자 접수함 문구는 공유확인과 검토대상을 분리하는 데 집중한다.

### 9-1. 탭 문구

```text
공유확인
검토 필요
조치 필요
조치완료
반려
전체
```

### 9-2. 상단 요약 문구

```text
오늘 공유확인
검토 필요
조치 필요
조치완료
```

### 9-3. 관리자 안내문

```text
공유확인과 위험제보를 구분해 확인하세요.
위험제보는 현장 확인 후 처리상태와 조치 메모를 남겨 주세요.
```

### 9-4. 조치 메모 안내

```text
조치 내용 또는 확인 결과를 입력해 주세요.
```

## 10. Status Copy

처리상태는 아래 5개를 기본으로 한다.

```text
접수
검토중
조치필요
조치완료
반려
```

상태 설명:

| 처리상태 | 설명                     |
| ---- | ---------------------- |
| 접수   | 제출 내용이 접수된 상태          |
| 검토중  | 관리자가 내용을 확인 중인 상태      |
| 조치필요 | 현장 조치 또는 추가 확인이 필요한 상태 |
| 조치완료 | 관리자가 조치 완료로 기록한 상태     |
| 반려   | 조치 대상이 아니거나 정보가 부족한 상태 |

주의:

```text
AI가 조치완료를 자동 확정하지 않는다.
조치완료는 관리자 확인 후 기록한다.
```

## 11. CEO / Customer Summary Copy

대표 또는 고객 담당자 요약 문구는 숫자 중심으로 표시한다.

```text
이번 달 공유확인 {count}건
위험제보 {count}건
아차사고 {count}건
개선제안 {count}건
미검토 {count}건
조치필요 {count}건
조치완료 {count}건
```

요약 문장 후보:

```text
이번 달 근로자 공유확인 {shareCount}건, 위험제보 {reportCount}건, 조치완료 {doneCount}건입니다.
미검토 {pendingCount}건은 확인이 필요합니다.
```

주의:

```text
공유확인 수를 조치완료 수로 표시하지 않는다.
```

## 12. Poster Copy

현장 QR 포스터 기본 문구는 아래를 우선 사용한다.

```text
위험요인 확인 / 의견제출 QR

오늘 작업 전 위험요인을 확인해 주세요.
의견이 없으면 “의견 없음”으로 제출해 주세요.
위험하거나 불편한 점이 있으면 사진과 함께 남겨 주세요.

1. QR 접속
2. 위험요인 확인
3. 의견 없음 또는 위험제보 제출
```

짧은형:

```text
위험요인 확인 / 의견제출 QR

오늘 작업 전 위험요인을 확인하고
의견이 있으면 남겨 주세요.
```

## 13. Industry Copy

업종별 QR 문구는 아래를 후보로 둔다.

### 13-1. 생활폐기물

```text
생활폐기물 수거 작업 전 위험요인 확인 QR

후진 차량, 적재함 협착, 미끄럼, 찔림·베임, 야간작업 위험을 확인해 주세요.
위험하거나 불편한 장소가 있으면 사진과 함께 남겨 주세요.
```

### 13-2. 물류

```text
물류 작업 전 위험요인 확인 QR

지게차 이동, 상·하차, 랙 주변, 고소대, 보행자 동선을 확인해 주세요.
개선이 필요한 점은 QR로 남겨 주세요.
```

### 13-3. 제조

```text
제조 작업 전 위험요인 확인 QR

끼임, 협착, 중량물, 감전, 고열, 지게차 위험을 확인해 주세요.
위험하거나 불편한 점은 QR로 남겨 주세요.
```

### 13-4. 화학물질 취급

```text
화학물질 취급 전 위험요인 확인 QR

물질명, 경고표지, 보호구, 환기, 비상대응 방법을 확인해 주세요.
이상 증상이나 누출 위험이 있으면 즉시 알리고 QR로 기록해 주세요.
```

주의:

```text
화학물질 취급 현장은 QR 제출보다 즉시 대피·보고·응급조치가 우선이다.
```

## 14. Weather Risk Copy

기상 위험 Level 2 이상일 때만 근로자 화면 또는 현장비서에 짧게 표시한다.

### 14-1. 폭염

```text
오늘은 온열질환 주의가 필요합니다.
물, 그늘, 휴식 준비상태를 확인해 주세요.
어지러움, 두통, 근육경련이 있으면 즉시 알리고 의견을 남겨 주세요.
```

체크 후보:

```text
온열질환 자각증상을 확인했습니다.
```

### 14-2. 강풍

```text
강풍 위험이 있는 작업은 작업 전 상태 확인이 필요합니다.
고소작업, 적재물, 외부 작업구역을 확인해 주세요.
```

주의:

```text
기상 위험 카드는 작업중지 명령이 아니다.
기상 위험 문구는 작업 전 확인 안내로 표시한다.
```

## 15. Legal Guard Copy

법적 주의 문구는 다음을 기본으로 한다.

```text
본 기능은 사업장의 위험성평가 공유, 근로자 확인, 의견제출, 관리자 검토 과정을 기록으로 남기기 위한 운영지원 기능입니다.
위험성평가의 최종 실시, 위험성 결정, 개선대책 확정, 현장 조치 실행 및 법적 책임은 사업주에게 있습니다.
```

짧은 안내:

```text
본 서비스는 법적 면책이나 과태료 방지를 보장하지 않습니다.
공유·확인·검토 과정을 기록으로 남기는 운영지원 도구입니다.
```

## 16. Forbidden Copy

코드와 문서에서 사용하지 않을 표현은 아래와 같다.

```text
위험성평가 법적 완료
근로자 참여 의무 자동 충족
법정교육 인정 완료
교육시간 인정 보장
과태료 방지
중대재해 면책
법적 방어 보장
무재해 보장
KOSHA 인정 보장
안전조치 완료 확정
QR 제출만으로 법적 의무 완료
AI가 조치완료를 판단
사진 업로드만으로 증빙 완료
```

구현 시 grep 확인 후보:

```text
위험성평가 법적 완료|과태료 방지|중대재해 면책|교육시간 인정 보장|KOSHA 인정 보장|무재해 보장|AI가 조치완료
```

## 17. Partner Demo Copy

Partner Demo는 실제 고객 데이터 없이 샘플 흐름만 보여준다.

샘플 문구:

```text
샘플 현장 위험요인을 확인해 주세요.
이 화면은 실제 고객 데이터가 아닌 체험용 샘플입니다.
```

샘플 제출 완료:

```text
샘플 확인 기록이 저장되었습니다.
Partner Demo에서는 실제 고객 DB에 저장되지 않습니다.
```

금지:

```text
실제 고객명
실제 고객 사진
실제 Notion DB
Supabase 직접 호출
API 직접 호출
Owner Console
토큰
API Key
환경변수 값
실제 고객 민감정보
```

## 18. Export Copy

고객용 Export에는 내부값을 표시하지 않는다.

Export 표시 후보:

```text
공유확인 목록
위험제보 목록
아차사고 목록
개선제안 목록
조치상태 목록
사진증빙 목록
월간 요약
```

Export 제외 문구:

```text
내부 UUID
raw payload
Notion URL
Owner/Admin 전용 링크
API Key
환경변수명과 값
토큰 또는 토큰 유사 문자열
내부 디버그 메시지
실제 고객 민감정보
```

## 19. TypeScript 상수 구조 후보

실제 구현 시 아래 형태를 후보로 둔다.

```ts
export const riskShareLinkCopy = {
  worker: {
    title: "위험요인 확인 / 의견제출",
    intro:
      "오늘 작업 전 위험요인을 확인해 주세요. 위험하거나 불편한 점이 있으면 의견을 남길 수 있습니다.",
    checks: {
      riskAssessment: "위험성평가 내용을 확인했습니다.",
      safetyMeasure: "오늘 안전조치와 주의사항을 확인했습니다.",
    },
```

## 24. 최종 정리

Risk Share Link Copy Constants Spec의 목적은 문구를 화면마다 흩어지게 만들지 않고, 코드에서 일관되게 관리하는 것이다.

최종 기준은 다음과 같다.

```text
근로자 화면은 짧게
관리자 화면은 상태 중심
대표 화면은 숫자 중심
포스터는 현장 친화적으로
법적 주의문은 보수적으로
금지표현은 코드 레벨에서 차단
업종별 문구는 별도 매핑
Partner Demo는 샘플 데이터만 사용
```

최종 한 줄:

```text
Risk Share Link 문구 상수화는 문서로 잠근 영업·운영 기준을 실제 앱 화면에 안전하게 연결하기 위한 중간 단계다.
```
