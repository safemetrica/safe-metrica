# SafeMetrica Risk Share Pack Sales Demo Capture Guide v1

기준일: 2026-06-14  
문서 성격: 영업 미팅용 캡처 순서 / 시연 안내 기준  
적용 대상: Risk Share Pack 제안, 우광개발 등 신규 고객 미팅

---

## 1. 목적

이 문서는 Risk Share Pack을 고객에게 설명할 때 어떤 화면을 어떤 순서로 보여줄지 정리한 캡처 가이드다.

목표는 기능을 많이 보여주는 것이 아니라, 아래 흐름을 3분 안에 이해시키는 것이다.

1. 위험성평가 자료를 근로자에게 공유한다.
2. 근로자가 확인하거나 의견을 낸다.
3. 근로자대표 참여확인도 별도 링크로 남긴다.
4. 관리자가 접수함에서 검토한다.
5. 월간요약과 Export로 운영기록을 남긴다.

---

## 2. 미팅에서 보여줄 화면 순서

권장 순서:

1. Risk Share Pack 관리자 홈
2. 근로자 QR 공유확인 화면
3. 위험제보 / 아차사고 / 개선제안 제출 화면
4. 근로자대표 참여확인 linkId 화면
5. 관리자 접수함
6. 월간보고서 Risk Share Pack 섹션
7. 고객용 CSV Export 설명
8. 고객 handoff guide 핵심 문구

---

## 3. 관리자 홈 캡처 항목

경로 후보:

- /manager/risk-share

캡처 포인트:

- 공유팩 운영 흐름이 보이는 첫 화면
- 근로자 공유확인 진입 안내
- 근로자대표 확인 링크 생성 또는 안내 영역
- 관리자 접수함으로 이동하는 흐름
- 월간보고서 또는 Export 연결 안내

설명 문장:

Risk Share Pack은 위험성평가표를 다시 작성하는 기능이 아니라, 작성 이후의 공유·확인·의견·검토·보고 흐름을 운영기록으로 남기는 구조입니다.

---

## 4. 근로자 QR 화면 캡처 항목

경로 후보:

- /field/participation
- /field/participation/risk-summary
- /field/participation/submitted

캡처 포인트:

- 앱 설치 없이 QR 또는 링크로 접근
- 위험요인 확인
- 안전조치 확인
- 공유확인 제출
- 제출 완료 화면

설명 문장:

근로자는 복잡한 앱을 쓰는 것이 아니라, QR로 들어와 오늘 공유된 위험요인과 안전조치를 확인하고 기록을 남깁니다.

---

## 5. 위험제보 제출 흐름 캡처 항목

캡처 포인트:

- 위험제보
- 아차사고
- 개선제안
- 익명 선택 가능 여부
- 사진 첨부 가능 여부
- 제출 완료 화면

설명 문장:

의견 없음은 공유확인으로 분리하고, 위험제보·아차사고·개선제안은 관리자 검토 대상으로 따로 분리합니다.

---

## 6. 근로자대표 참여확인 캡처 항목

경로 후보:

- /field/representative-confirmation?linkId=...

캡처 포인트:

- linkId 기반 외부 링크
- URL에 회사코드, 현장명, 확인범위 직접 노출 없음
- 근로자대표 성명 / 소속 / 의견 입력
- 개인정보 안내 확인
- 제출 완료

설명 문장:

근로자대표 참여확인은 일반 제보와 분리해 별도 원장으로 남기고, 외부 링크에는 내부 회사코드나 토큰을 직접 노출하지 않습니다.

---

## 7. 관리자 접수함 캡처 항목

경로 후보:

- /manager/representative-confirmations
- /manager/risk-share

캡처 포인트:

- 공유확인과 제보 분리
- 근로자대표 참여확인 분리
- 처리상태
- 조치메모
- 검토 필요 항목

설명 문장:

공유확인은 교육·주지 확인 흐름으로 보고, 위험제보와 개선제안은 조치 검토 흐름으로 따로 봅니다. 이 분리가 대표 지표 왜곡을 막습니다.

---

## 8. 월간보고서 캡처 항목

경로 후보:

- /monthly-report/risk-share
- /monthly-report

캡처 포인트:

- 공유확인 건수
- 제보 / 아차사고 / 개선제안 건수
- 조치필요 / 조치완료 후보
- 근로자대표 참여확인
- 다음 평가주기 반영 후보

설명 문장:

월간보고서는 법적 적합성 판단서가 아니라, 고객의 안전보건 관리·확인 과정이 어떻게 운영됐는지 정리하는 운영요약입니다.

---

## 9. 고객용 CSV Export 설명 항목

경로 후보:

- /owner 내부 Export 패널
- /api/admin/export/customer-csv

주의:

Owner 화면은 고객 미팅에서 직접 열어 보여주지 않는다. 필요한 경우 캡처 또는 설명자료로만 안내한다.

설명 문장:

CSV Export는 고객이 직접 내부 원장을 내려받는 구조가 아니라, 내부 운영자가 고객 전달용 필드만 정제해 제공합니다.

---

## 10. 절대 보여주면 안 되는 화면

미팅이나 외부 캡처에서 아래 항목은 보여주지 않는다.

- Owner Console 실화면
- Owner Token
- Supabase 원장 직접 화면
- 환경변수
- API Key
- service role
- 실제 고객 데이터
- 실제 고객 사진
- 내부 디버그 메시지
- 토큰이 포함된 URL
- Vercel 환경변수 화면
- GitHub secret 화면

---

## 11. 우광개발 미팅용 3분 설명 순서

1분차:

위험성평가표는 이미 작성돼 있어도, 근로자에게 실제 공유됐는지와 의견이 있었는지 기록이 약한 경우가 많습니다.

2분차:

Risk Share Pack은 QR로 위험요인을 공유하고, 근로자 확인·위험제보·근로자대표 참여확인을 기록으로 남깁니다.

3분차:

관리자는 접수함에서 검토하고, 월간요약과 Export로 운영기록을 정리합니다. 단, 이 시스템은 위험성평가 대행이나 법적 보장을 제공하는 서비스가 아닙니다.

---

아래 표현은 영업자료, 미팅 설명, 캡처 주석에 사용하지 않는다.

- 과태료 방지
- 중대재해 면책
- 무재해 보장
- 위험성평가 대행
- 안전관리대행
- AI가 법적 판단
- AI가 조치완료 확정
- QR 제출만으로 법적 의무 완료

## 14. 버전 기준

v1은 영업 미팅용 캡처 순서 문서다.

실제 고객 데이터 캡처는 사용하지 않는다.  
Partner Demo 또는 샘플 데이터 화면을 우선 사용한다.  
고객별 운영 화면은 계약 후 내부 세팅 완료 뒤 별도 캡처한다.
