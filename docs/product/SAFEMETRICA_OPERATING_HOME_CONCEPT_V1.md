# SafeMetrica Operating Home Concept v1

## 1. 결론

SafeMetrica 홈은 일반 홈페이지가 아니라 안전운영 상황판이다.

처음 컬러 카드형 홈은 데모, 소개, 파트너 설명용 첫인상으로는 좋다.
하지만 실제 고객이 매일 쓰는 홈은 기능 전시장이 아니라 오늘 운영상태, 확인 신호, 남은 기록, 받을 자료를 보여주는 운영판이어야 한다.

## 2. 홈 분리 원칙

SafeMetrica 홈은 한 화면이 모든 역할을 하지 않는다.

### 브랜드/데모 홈

용도:

- 파트너 데모
- 영업 미팅
- 제품 첫인상
- SafeMetrica 핵심 기능 설명

방향:

- 컬러 카드형 구조 유지 가능
- TBM, 증빙, PTW, 대시보드 등 핵심 흐름을 간단히 보여준다.
- 실제 고객 데이터, Owner Console, 토큰, API Key, 환경변수 값은 노출하지 않는다.

### 실제 운영 홈

용도:

- 실제 고객이 매일 보는 화면
- 오늘 할 일과 확인 신호 중심
- 기록이 남고 있는지 확인

방향:

- 기능 목록보다 운영 흐름을 먼저 보여준다.
- 오늘 현장 운영상태, 미처리 항목, 증빙 보완, 월간보고서, 받을 자료를 중심으로 구성한다.

### Owner 홈

용도:

- 내부 운영자 전용
- 고객사 세팅, 원장, Export, 최종 공유본, 백업 관리

방향:

- 전체 기능을 볼 수 있어도 된다.
- 외부 고객에게 공유하지 않는다.

## 3. 데스크탑과 모바일 분리

### 모바일

모바일은 실행형이다.

근로자:

- QR 또는 링크 진입
- 오늘 위험 확인
- 확인했습니다
- 제보하기
- 제출 완료

현장관리자:

- 말로 TBM 작성
- 오늘 TBM 작성하기
- 작성 내역 확인
- 제보 확인
- 조치 필요 확인
- 증빙 추가

### 데스크탑

데스크탑은 운영 상황판이다.

우선 표시:

- 오늘 운영 상태
- 오늘 먼저 볼 신호
- 미처리 항목
- 증빙 보완
- 월간보고서
- 받을 자료

## 4. 현장관리자 운영 홈

현장관리자 홈은 오늘 처리 순서 중심이다.

권장 구조:

1. 오늘 안전운영 상태
2. 오늘 먼저 볼 신호
3. 오늘 처리 순서
4. TBM 빠른 실행
5. 이번 달 기록 상태
6. 전체 기능 보기

예시:

오늘 안전운영

상태: 확인 필요

오늘 먼저 볼 신호

- TBM 작성 필요
- 위험제보 미검토
- 조치 필요 항목
- 증빙 보완 필요
- PTW 검토 후보

오늘 처리 순서

1. TBM 작성
2. 근로자 확인·제보 확인
3. 조치 기록
4. 증빙 보완
5. 월간보고서 반영

빠른 실행

- 말로 TBM 작성
- 오늘 TBM 작성하기
- 작성 내역 확인

전체 기능 메뉴는 기본 접힘으로 둔다.

## 5. 대표 운영 홈

대표 홈은 입력 화면이 아니라 운영 브리핑이다.

대표가 볼 항목:

- 오늘 운영 상태
- 미조치 항목
- 반복 위험
- 근로자 제보 요약
- 증빙 누락 요약
- 위험성평가 공유 현황
- 월간보고서 준비 상태
- 대표 확인 필요 항목
- 받을 자료

대표에게는 현장비서가 아니라 대표 운영 브리핑을 제공한다.

## 6. 근로자 화면

근로자는 복잡한 홈이 필요 없다.

근로자 흐름:

1. QR 또는 링크 접속
2. 오늘 위험 확인
3. 위험성평가 공유확인
4. TBM 확인
5. 제보 또는 의견 제출
6. 제출 완료

근로자에게 접수함, 처리상태 전체, 대표 요약, Export, Owner Console을 노출하지 않는다.

## 7. Owner 화면

Owner는 내부 운영자 전용이다.

Owner에서 관리할 항목:

- 고객사 선택
- 테넌트 세팅
- 위험성평가 공유팩 후보 검토
- 고객 확인 상태
- 최종 공유본 확정
- 근로자 QR 노출 상태
- 월별 보관함
- 고객 전달 자료
- 원장 Export
- 내부 백업

외부 공유 금지:

- Owner Console
- token
- API Key
- 환경변수 값
- service role
- raw_payload
- 내부 UUID
- 실제 고객 민감정보

## 8. 홈에서 강조할 단어

고객 화면에서는 기능명보다 운영 언어를 우선한다.

권장 표현:

- 오늘 운영상태
- 오늘 확인 신호
- 증빙 현황
- 증빙 보관함
- 월별 보관함
- 고객 전달 자료
- 받을 자료
- 대표 운영 브리핑
- 위험성평가 공유팩
- 최종 공유본 확정

## 9. 전체 기능 메뉴

전체 기능 메뉴는 필요하지만 홈의 주인공이 아니다.

기준:

- 기본 접힘
- 필요 시 전체 기능 보기로 펼침
- Owner는 전체 기능 접근 가능
- 실제 고객 홈에서는 오늘 운영 흐름을 우선 표시

## 10. 금지 표현

아래 표현은 홈, 제안서, 랜딩, 보고서에서 사용하지 않는다.

- 법적 완료
- 면책 보장
- 과태료 방지
- 무재해 보장
- 안전관리대행
- 위험성평가 대행 대체
- AI가 조치완료 확정
- AI가 법적 판단
- PTW 자동 승인
- KOSHA 인정 보장
- 교육시간 인정 보장

## 11. 후속 PR 순서

1. 현장관리자 홈 상단을 오늘 운영상태 중심으로 재배치
2. 전체 기능 메뉴 기본 접힘 처리
3. 대표 홈을 대표 운영 브리핑 중심으로 재배치
4. 증빙 보관함, 월별 보관함, 받을 자료 영역 추가
5. Work Signal을 오늘 확인 신호로 통합
6. 말로 PTW를 고위험작업허가 안의 보조 CTA로 연결
