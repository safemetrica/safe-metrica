# SafeMetrica Risk Share Evidence Pack Productization Lock v1

## 1. 목적

이 문서는 SafeMetrica 위험성평가 공유확인 증빙팩, 이하 위공팩, 의 상품화 기준을 고정한다.

이번 기준은 2026-07-01 업로드된 위공팩 IP 초안, CURRENT OPERATION LOCK, COPY GUARD, STARTUP PROCESS LOCK, PRODUCT ROADMAP LOCK을 바탕으로 정리한다.

이번 문서는 다음을 하지 않는다.

- 저장 API 생성
- DB insert
- RLS policy 생성
- 기존 고객 route 변경
- 고객 데이터 query 연결
- tenant onboarding write flow 진행
- user invite/email flow 연결
- 법적 판단 또는 조치 확정 표현 사용

## 2. 제품 정의

위공팩은 위험성평가표를 대신 작성하는 서비스가 아니다.

위공팩은 작성된 위험요인이 현장에서 공유되고, 자사 근로자와 외부 인력이 확인하며, 익명 의견·위험제보와 관리자 검토, 조치메모, 추적 필요 항목이 월간 안전운영 기록 후보로 정리되도록 돕는 산업안전 운영기록 SaaS 상품이다.

고객 설명 기준:

> 위험성평가표만 갖고는 부족합니다. 현장에서 실제로 공유되고 확인되었는지, 의견을 낼 수 있었는지, 관리자가 검토했는지 운영기록으로 남아야 합니다.

위공팩은 Full SafeMetrica의 진입 상품이다.

Full SafeMetrica, Document Studio, PTW, TBM History, Daily Safety State, AI 운영브리핑은 확장 방향으로 분리한다.

## 3. IP 초안 구조와 제품 구조 매핑

업로드된 위공팩 IP 초안의 핵심 흐름은 제품 구조와 일치한다.

제품 매핑:

| IP 초안 구성 | SafeMetrica 제품 구조 |
| --- | --- |
| 관리자 단말 | 관리자/Owner 위험요인 공유본 등록 화면 |
| 위험성평가 공유본 관리부 | Version Lock 공유본 / 공유 위험요인 목록 |
| QR 확인 처리부 | 근로자·외부인력 QR 진입 |
| 실명 확인기록 처리부 | 실명 확인·소속 확인·모바일 서명 |
| 익명 의견 처리부 | 익명 의견·위험제보 별도 flow |
| 관리자 검토부 | 관리자 접수함·검토상태·조치메모 |
| 안전운영 원장 | Supabase 운영 원장 / legacy bridge 보호 |
| 월간 증빙팩 생성부 | 월간 안전운영 결과물 후보 |

단, 고객 화면과 제안서에는 내부 기술 용어를 노출하지 않는다.

고객용 표현:

- 안전운영 원장
- 공유확인 기록
- 익명 의견
- 관리자 검토
- 월간 안전운영 결과물
- 확인자료 정리
- 추적 필요 항목

고객용 금지 표현:

- Supabase
- Notion bridge
- raw_payload
- schema
- service role
- internal API
- Owner Token
- RLS policy name

## 4. 실명 확인·서명 flow

실명 확인 flow는 위험요인 공유본 또는 공유 위험요인을 실제로 확인한 사람의 확인기록을 남기는 흐름이다.

포함 후보:

- 이름
- 소속 또는 운영상 분류
- 휴대폰 뒷자리 또는 사번 등 고객별 확인 보조값
- 확인 시각
- 위험요인 공유본 또는 버전 식별자
- 모바일 자필서명
- 확인 문구 동의

원칙:

- 실명 확인 flow와 익명 의견 flow는 분리한다.
- 확인기록은 공유본 또는 공유 기준일에 연결한다.
- 근로자와 외부 인력은 운영상 분류로 구분할 수 있다.
- 모바일 UI는 짧은 문장, 큰 버튼, 명확한 확인 흐름을 우선한다.
- 리치 납품형 Light UI는 근로자 모바일 UX 패턴만 공통 Core로 추출한다.

금지:

- 익명 의견 flow에서 이름, 소속, 확인번호, 서명을 받지 않는다.
- 실명 확인과 익명 의견을 한 화면에서 섞어 저장하지 않는다.
- 확인 버튼 클릭만으로 위험요인 공유확인을 완료하지 않는다.
- 위험요인 목록이 없는 상태에서 월간 공유확인을 진행시키지 않는다.

## 5. 익명 의견·위험제보 flow

익명 의견 flow는 위험요인에 대한 의견, 아차사고, 개선제안, 위험제보를 식별정보 없이 접수하는 별도 흐름이다.

포함 후보:

- 의견 유형
- 의견 내용
- 긴급도 또는 참고 여부
- 관련 작업구역 후보
- 첨부 사진 후보
- 제출 시각
- 공유본 또는 공유 기준일 연결 후보

원칙:

- 이름, 소속, 확인번호, 자필서명은 수집하지 않는다.
- 실명 확인 테이블 또는 확인기록과 물리적·논리적으로 분리한다.
- 관리자 검토 전에는 조치 완료로 표시하지 않는다.
- AI 또는 자동 요약은 의견 분류와 요약 후보만 제공한다.
- 최종 검토와 조치 판단은 관리자 또는 사업주가 수행한다.

금지:

- 완전 익명 보장
- 제보자 신원 완전 차단 보장
- 사고 예방 보장
- 법적 책임 면제
- AI가 조치 필요 여부를 확정 판단

## 6. 외부 인력 확인 flow

위공팩의 핵심 구매 포인트 중 하나는 외부 인력 확인 흐름이다.

운영상 분류 후보:

- 협력사
- 도급사
- 파견
- 납품
- 상하차
- 정비업체
- 방문자
- 기타 외부 인력

설명 기준:

외부 인력 구분은 법적 지위 판단이 아니라 현장 안전공유와 확인 기록을 위한 운영상 분류다.

기록 후보:

- 외부 인력 유형
- 소속명 또는 업체명
- 확인자 이름
- 확인 시각
- 확인한 위험요인 공유본
- 서명 여부
- 관리자 검토 필요 여부

금지:

- 도급관계 법적 판단
- 근로자성 판단
- 원청 책임 범위 판단
- 법적 지위 자동 분류
- 외부 인력 안전관리 완료 보장

## 7. 관리자 검토 flow

관리자 검토 flow는 접수된 확인기록, 미확인 항목, 익명 의견, 조치메모, 추적 필요 항목을 검토하는 흐름이다.

포함 후보:

- 미확인자 확인
- 익명 의견 검토
- 위험제보 검토
- 조치메모 작성
- 보완요청
- 추적 필요 여부
- 다음 달 이월 여부
- 월간보고서 반영 후보

원칙:

- 접수됨과 조치 완료를 구분한다.
- 관리자 검토 전에는 자동으로 조치 완료 처리하지 않는다.
- 월간 결과물은 관리자 검토 후보와 확정 기록을 구분한다.
- 고객에게는 내부 테이블이나 API 구조를 노출하지 않는다.

## 8. 월간 안전운영 증빙팩 후보

월간 안전운영 증빙팩은 위공팩의 핵심 산출물이다.

구성 후보:

- 공유 위험요인 요약
- 공유본 기준일 / 버전 식별자
- 근로자 확인 현황
- 외부 인력 확인 현황
- 미확인 항목
- 익명 의견·위험제보 요약
- 관리자 검토 현황
- 조치메모
- 추적 필요 항목
- 다음 달 보완 후보
- 대표/관리자 브리핑 문구

분석 문구 예시:

> 이번 달에는 출고장 지게차 운행, 성형기 주변 협착, 적재·하역 동선 관련 위험요인이 공유되었습니다. 자사 근로자와 외부 인력의 확인기록이 구분되어 저장되었고, 익명 의견 중 일부는 관리자 검토 및 다음 달 추적 필요 항목으로 분류되었습니다.

표현 기준:

- 월간 안전운영 결과물
- 월간 운영기록 후보
- 관리자 검토 후 확정
- PDF 저장/인쇄 후보
- 확인자료 정리

금지:

- 법적 증빙 완료
- 과태료 방지
- 중대재해처벌 면책
- KOSHA 인정 보장
- 위험성평가 완료 판단
- 안전조치 완료 확정
- 사고 예방 보장

## 14. 현재 결론

#758까지 SaaS onboarding 준비 작업은 일시 정지선에 도달했다.

이제 우선순위는 위공팩 상품화다.

위공팩은 SafeMetrica의 첫 상용화 진입 상품이며, 위험성평가표를 대신 작성하는 서비스가 아니라 위험요인 공유, 근로자·외부 인력 확인, 익명 의견, 관리자 검토, 조치메모, 월간 안전운영 결과물 후보를 연결하는 산업안전 운영기록 SaaS 상품이다.

이번 주는 위공팩을 제안서, 월간 결과물 샘플, 비식별 데모, 모바일 QR 공통 UX까지 팔 수 있는 상태로 정리한다.
