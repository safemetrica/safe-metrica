# SafeMetrica Risk Share Monthly Result Sample Lock v1

## 1. 목적

이 문서는 SafeMetrica 위험성평가 공유확인 증빙팩, 이하 위공팩, 의 월간 안전운영 결과물 샘플 구조를 고정한다.

위공팩은 QR 확인 기능 자체가 아니라, 한 달 동안 위험요인 공유, 근로자 확인, 외부 인력 확인, 익명 의견, 관리자 검토, 조치메모, 추적 필요 항목이 어떻게 운영되었는지를 고객이 확인할 수 있는 월간 결과물 체감 상품이다.

이번 문서는 다음을 하지 않는다.

- 월간보고서 route 코드 변경
- PDF 생성 API 변경
- DB query 변경
- 실제 고객 데이터 사용
- 기존 고객 route 변경
- 법적 판단 또는 조치 확정 표현 사용

## 2. 월간 결과물의 위치

월간 결과물은 위공팩의 핵심 산출물이다.

고객은 단순히 QR을 찍었다는 사실보다 다음을 확인하고 싶어 한다.

- 어떤 위험요인이 공유되었는가
- 자사 근로자가 확인했는가
- 외부 인력이 확인했는가
- 의견이나 위험제보가 있었는가
- 관리자가 검토했는가
- 조치메모가 남았는가
- 다음 달 추적 필요 항목이 있는가

따라서 위공팩 설명은 QR 기능 중심이 아니라 월간 결과물 중심으로 구성한다.

## 3. 월간 결과물 제목 후보

고객-facing 제목 후보:

- 월간 안전운영 결과
- 위험성평가 공유확인 월간 운영기록
- 공유·확인·의견·검토 월간 요약
- Risk Share Evidence Pack 월간 결과물

추천 제목:

> 위험성평가 공유확인 월간 안전운영 결과

보조 제목:

> 위험요인 공유, 근로자·외부 인력 확인, 의견·제보, 관리자 검토 흐름을 정리한 월간 운영기록입니다.

## 4. 월간 결과물 구성

기본 구성:

1. 대표 요약
2. 공유본 정보
3. 공유 위험요인 요약
4. 근로자 확인 현황
5. 외부 인력 확인 현황
6. 미확인 항목
7. 익명 의견·위험제보 접수 현황
8. 관리자 검토 현황
9. 조치메모
10. 다음 달 추적 필요 항목
11. 운영 브리핑 후보
12. 관리자 확인 안내

## 5. 대표 요약 샘플

샘플 문구:

> 이번 달에는 출고장 지게차 운행, 성형기 주변 협착, 적재·하역 동선 관련 위험요인이 공유되었습니다. 자사 근로자와 외부 인력의 확인기록이 구분되어 저장되었고, 익명 의견 중 일부는 관리자 검토 및 다음 달 추적 필요 항목으로 분류되었습니다.

주의:

- “위험성평가 완료”라고 쓰지 않는다.
- “법적 증빙 완료”라고 쓰지 않는다.
- “과태료 방지”라고 쓰지 않는다.
- “사고 예방”을 보장하지 않는다.
- “AI가 판단했다”고 쓰지 않는다.

## 7. 공유 위험요인 요약 샘플

표시 항목 후보:

| 구분 | 위험요인 | 작업구역 | 확인 포인트 |
| --- | --- | --- | --- |
| 지게차 | 보행자 충돌 | 출고장 | 동선 분리, 후진 시 유도 확인 |
| 기계 | 협착 | 성형실 | 방호장치, 손 끼임 주의 |
| 적재 | 낙하·전도 | 적재장 | 적재 높이, 팔레트 상태 |
| 하역 | 끼임·충돌 | 하역 구역 | 작업 반경 확인 |
| 온열 | 고온 노출 | 전 작업구역 | 수분섭취, 휴식 |

문구 기준:

> 본 항목은 공유된 위험요인의 요약입니다. 실제 작업별 위험성평가 내용과 현장 조치 여부는 관리자가 별도로 검토합니다.

## 8. 근로자 확인 현황 샘플

표시 항목 후보:

| 구분 | 대상 | 확인 | 미확인 | 비고 |
| --- | ---: | ---: | ---: | --- |
| 자사 근로자 | 42명 | 39명 | 3명 | 미확인자 후속 확인 필요 |
| 현장관리자 | 3명 | 3명 | 0명 | 완료 |
| 합계 | 45명 | 42명 | 3명 | 확인률은 운영 참고값 |

분석 문구:

> 자사 근로자 대부분이 공유 위험요인을 확인했으나, 일부 미확인 항목은 다음 현장회의 또는 작업 전 확인 시 후속 안내가 필요합니다.

주의:

- 확인률을 법적 충족률처럼 표현하지 않는다.
- 100% 충족, 완전 이행 같은 표현을 쓰지 않는다.

## 9. 외부 인력 확인 현황 샘플

표시 항목 후보:

| 운영상 분류 | 확인 건수 | 주요 작업 | 비고 |

금지:

- 도급관계 법적 판단
- 원청 책임 판단
- 외부 인력 안전관리 완료 보장

## 10. 익명 의견·위험제보 접수 현황 샘플

표시 항목 후보:

| 유형 | 건수 | 관리자 검토 | 추적 필요 |
| --- | ---: | ---: | ---: |
| 위험제보 | 2건 | 2건 | 1건 |
| 아차사고 | 1건 | 1건 | 0건 |
| 개선요청 | 3건 | 2건 | 2건 |
| 기타 의견 | 1건 | 1건 | 0건 |

분석 문구:

> 익명 의견은 총 7건 접수되었으며, 이 중 3건은 다음 달 추적 필요 후보로 분류되었습니다. 관리자는 접수 내용을 검토하고 필요한 경우 조치메모 또는 보완 요청을 남깁니다.

주의:

- 익명 의견으로 제출자 신원 추정 문구를 만들지 않는다.
- 완전 익명 보장 표현을 쓰지 않는다.
- 의견 접수만으로 조치 완료 처리하지 않는다.

## 11. 관리자 검토 현황 샘플

표시 항목 후보:

| 상태 | 건수 | 설명 |
| --- | ---: | --- |
| 검토 완료 | 5건 | 관리자가 내용을 확인하고 메모를 남긴 항목 |
| 조치메모 작성 | 3건 | 현장 조치 또는 후속 확인 메모가 있는 항목 |
| 추적 필요 | 3건 | 다음 달 재확인 후보 |
| 미검토 | 2건 | 관리자 확인 필요 |

분석 문구:

> 관리자 검토가 완료된 항목은 월간 결과물에 반영되며, 미검토 항목은 다음 점검 또는 회의에서 확인할 필요가 있습니다.

금지:

- 관리자가 검토하지 않았는데 완료 표시
- AI가 조치 완료 확정
- 시스템이 안전조치 완료 판단

## 17. PDF / Export 기준

월간 결과물은 고객이 확인할 수 있어야 한다.

출력 후보:

- PDF 저장
- 인쇄
- 내부 운영자 Export
- 고객 전달용 요약본
- 관리자 보관용 상세본

주의:

- CSV 원본은 고객 직접 다운로드보다 내부 운영자가 검토 후 전달하는 방식이 기본이다.
- 고객 민감정보와 내부 schema, raw payload, API path는 출력하지 않는다.
- 익명 의견 원문은 고객 제공 범위와 비식별 기준을 별도 검토한다.

월간 결과물은 위험요인 공유, 근로자 확인, 외부 인력 확인, 익명 의견, 관리자 검토, 조치메모, 추적 필요 항목을 고객이 확인할 수 있게 정리해야 한다.

다음 코드 작업은 monthly-report/risk-share 화면에 이 샘플 브리핑 구조를 반영하는 작은 read-only polish PR로 진행한다.
