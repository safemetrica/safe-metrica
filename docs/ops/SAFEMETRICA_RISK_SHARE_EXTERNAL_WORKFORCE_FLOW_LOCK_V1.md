# SafeMetrica Risk Share External Workforce Flow Lock v1

## 1. 목적

이 문서는 SafeMetrica 위험성평가 공유확인 증빙팩, 이하 위공팩, 에서 외부 인력 확인 흐름을 어떻게 설명하고 상품화할지 고정한다.

외부 인력 확인 흐름은 위공팩의 핵심 구매 포인트 중 하나다.

단, 이 흐름은 법적 지위 판단, 도급관계 판단, 근로자성 판단, 원청 책임 판단을 대신하지 않는다.

이번 문서는 다음을 하지 않는다.

- route 변경
- DB schema 변경
- submit API 변경
- tenant write/API 생성
- RLS policy 생성
- 기존 고객 route 변경
- 법적 판단 표현 사용

## 2. 외부 인력 확인 흐름 정의

외부 인력 확인 흐름은 사업장에 출입하거나 작업에 관여하는 외부 인력이 현장 위험요인 공유본을 확인하고, 운영상 분류와 확인기록을 남기는 흐름이다.

고객 설명 기준:

> 외부 인력 확인은 법적 지위 판단이 아니라, 현장에서 어떤 외부 인력이 어떤 위험요인을 공유받고 확인했는지 운영기록으로 남기기 위한 흐름입니다.

## 3. 운영상 분류 후보

운영상 분류 후보:

- 협력사
- 도급사
- 파견
- 납품
- 상하차
- 정비업체
- 방문자
- 기타 외부 인력

주의:

- 위 분류는 현장 안전공유와 확인기록을 위한 운영상 분류다.
- 법적 지위, 근로자성, 도급관계, 원청 책임 범위를 판단하지 않는다.
- 고객별 실제 분류명은 계약 범위와 현장 운영 기준에 따라 조정한다.

## 4. 외부 인력 확인 기록 후보

기록 후보:

- 외부 인력 유형
- 소속명 또는 업체명
- 확인자 이름
- 연락처 일부 또는 확인 보조값
- 확인 시각
- 확인한 위험요인 공유본
- 확인 기준일 또는 버전 식별자
- 모바일 서명 여부
- 관리자 검토 필요 여부

금지 저장값:

- 주민등록번호
- 비밀번호
- 내부 인증값
- API key
- service role
- Owner Token
- 고객 민감자료 원문
- 법적 지위 판단 결과

## 5. 근로자 확인 flow와의 차이

근로자 확인 flow:

- 자사 근로자 또는 내부 구성원의 공유확인 중심
- 이름, 소속, 확인 보조값, 모바일 서명 가능
- 월간 근로자 확인 현황으로 정리

외부 인력 확인 flow:

- 외부 방문·작업·납품·정비 등 운영상 분류 중심
- 소속 또는 방문 목적을 확인 후보로 둠
- 월간 외부 인력 확인 현황으로 별도 정리

공통 원칙:

- 위험요인 공유본 또는 버전 식별자와 연결한다.
- 확인 시각을 남긴다.
- 관리자 검토 후보로 연결할 수 있다.
- 확인기록은 조치 완료나 법적 충족 판단이 아니다.

## 6. 익명 의견 flow와의 분리

외부 인력도 의견이나 위험제보를 남길 수 있다.

다만 실명 확인 flow와 익명 의견 flow는 분리한다.

실명 확인 flow:

- 이름, 소속, 확인 보조값, 서명 가능
- 확인기록으로 저장

익명 의견 flow:

- 이름, 소속, 확인번호, 서명 수집 금지
- 의견, 위험제보, 아차사고, 개선요청 중심
- 관리자 검토 후보로 저장

금지:

- 외부 인력 실명 확인과 익명 의견을 한 화면에서 섞어 저장
- 익명 의견에 서명 요구
- 익명 의견 제출자를 추정하는 문구
- 완전 익명 보장 표현

## 7. 모바일 화면 문구 기준

외부 인력 확인 화면 문구 후보:

> 방문 또는 작업 전 공유된 위험요인을 확인해주세요.

> 소속 또는 방문 목적을 선택하고 확인기록을 남겨주세요.

> 이 분류는 법적 지위 판단이 아니라 현장 안전공유 기록을 위한 운영상 구분입니다.

확인 버튼 문구 후보:

- 위험요인 확인 후 서명
- 공유내용 확인
- 확인기록 남기기

주의 문구:

> 긴급한 위험이 있거나 작업 중 즉시 조치가 필요한 경우에는 현장 관리자에게 바로 알려주세요.

## 8. 월간 결과물 표시 기준

월간 안전운영 결과물에는 외부 인력 확인 현황을 근로자 확인 현황과 구분해 표시한다.

표시 후보:

| 운영상 분류 | 확인 건수 | 주요 작업 | 비고 |
| --- | ---: | --- | --- |
| 납품 | 8건 | 원자재 입고 | 입고 동선 안내 |
| 상하차 | 6건 | 제품 출고 | 지게차 동선 확인 |
| 정비업체 | 2건 | 설비 점검 | 작업구역 출입 전 확인 |
| 방문자 | 3건 | 현장 방문 | 안내자 동행 |

분석 문구 후보:

> 외부 인력 확인기록은 납품, 상하차, 정비업체, 방문자 등 운영상 분류로 정리되었습니다. 이 분류는 법적 지위 판단이 아니라 현장 안전공유와 확인기록을 위한 운영상 구분입니다.

## 9. 관리자 검토 기준

외부 인력 확인 흐름에서 관리자가 확인할 항목:

- 확인기록 누락 여부
- QR 안내문 배포 여부
- 반복 방문 외부 인력 확인 여부
- 위험요인 공유본 버전 일치 여부
- 외부 인력 의견·제보 접수 여부
- 다음 달 추적 필요 여부

관리자 문구:

> 외부 인력 확인기록은 현장 안전공유 흐름을 점검하기 위한 운영자료입니다. 법적 판단이나 조치 완료 판단을 대신하지 않습니다.

## 10. 고객-facing 금지 표현

사용하지 않는다.

- 도급관계 판단
- 근로자성 판단
- 원청 책임 판단
- 외부 인력 안전관리 완료
- 법적 책임 면제
- 중대재해처벌 대응 완료
- 과태료 방지
- 안전조치 완료 보장
- 무재해 보장
- AI 법적 판단

## 11. 사용 권장 표현

우선 사용한다.

- 외부 인력 운영상 분류
- 현장 안전공유 확인기록
- 방문·작업 전 위험요인 확인
- 월간 외부 인력 확인 현황
- 관리자 검토 후보
- 운영기록
- 확인자료 정리
- 최종 판단은 관리자 또는 사업주 수행

## 12. 기존 고객 보호

기존 고객은 외부 인력 확인 흐름으로 강제 전환하지 않는다.

보호 대상:

- daedo
- dongwoo
- hankookgreen
- bubblemon

기준:

- 기존 route 유지
- 기존 Field QR link 유지
- 기존 submit API 유지
- legacy Notion bridge 급작 제거 금지
- 로그인 강제 전환 금지
- 필요한 경우 targeted fix만 진행

## 13. 구현 우선순위 후보

권장 순서:

1. 본 문서로 외부 인력 확인 흐름 기준 잠금
2. 월간 결과물 화면에 외부 인력 확인 현황 샘플 섹션 보강
3. field participation 화면에서 실명 확인과 익명 의견 분리 기준 재확인
4. 신규 고객용 공통 Field QR Core 설계
5. 실제 외부 인력 submit 구조는 별도 inspect 후 진행

## 14. 현재 결론

외부 인력 확인 흐름은 위공팩의 핵심 상품성 요소다.

다만 법적 지위 판단으로 오해되지 않도록 운영상 분류, 확인기록, 관리자 검토, 월간 결과물 중심으로 설명한다.

다음 구현은 route/API 변경이 아니라 월간 결과물 화면의 외부 인력 확인 현황 샘플 섹션 보강부터 진행한다.
