# SafeMetrica Richi Tenant Setup Checklist v1

## 1. 목적

이 문서는 리치코리아 1주일 체험판을 운영환경에서 확인하기 위한 tenant 세팅 체크리스트다.

리치코리아 worker copy는 코드에 추가되어도, 운영환경에서는 Companies DB에 고객사 row가 등록되어 있어야 실제 tenant로 동작한다.

따라서 이 문서는 리치코리아 전용 QR 전자확인·피드백 화면을 열기 전 필요한 Companies DB 세팅, 확인 순서, 금지사항을 잠그기 위한 기준이다.

## 2. 전제

운영환경에서는 tenant fallback을 사용하지 않는다.

회사코드가 없거나 Companies DB에서 active row를 찾지 못하면 고객사 데이터 노출 방지를 위해 실패해야 한다.

리치코리아의 기본 회사코드는 아래 기준으로 둔다.

    companyCode=richi

허용 alias 후보:

- richi-korea
- richi_korea
- richikorea
- rich

위 alias는 화면 copy helper에서 richi로 정규화한다. 운영 Companies DB의 기준값은 richi로 통일한다.

## 3. Companies DB 등록 체크리스트

Companies DB에 리치코리아 row를 추가하거나 확인한다.

필수 확인 항목:

1. Company
   - 표시명: 리치코리아

2. companyCode
   - 값: richi
   - 대소문자와 공백 혼입 금지

3. active
   - true
   - 체험판 종료 또는 중단 시 false 전환 가능

4. tbmDbId
   - 1주일 체험판에서 TBM 기능을 바로 쓰지 않더라도 필드 존재 여부 확인
   - 실제 DB ID 값은 문서에 기록하지 않는다

5. ebmDbId
   - 1차 체험판 범위에서는 필수 기능이 아니지만 Companies DB 필드 구조상 확인
   - 실제 DB ID 값은 문서에 기록하지 않는다

6. ptwDbId
   - 1차 체험판 범위에서는 제외
   - 필드가 필요하면 빈 값 또는 운영 기준에 맞게 별도 판단

7. fieldVoiceDbId 또는 현장참여 DB 연결 필드
   - QR 전자확인·현장 피드백 제출 저장에 필요한 핵심 항목
   - 실제 DB ID 값은 문서에 기록하지 않는다

8. monthlyReportDbId 또는 보고서 관련 필드
   - 1주일 체험판에서는 정식 PDF 보고서 자동생성 제외
   - 주간 요약 copy 확인용으로 별도 문서화 가능

9. riskAssessmentDbId
   - 위험성평가 공유 항목을 노출할 경우 필요
   - 1주일 체험판에서 위험성평가 공유를 쓰지 않는다면 QR 전자확인·피드백 중심으로 제한

## 4. 1주일 체험판 기능 범위 확인

리치코리아 체험판은 아래 기능만 1차로 확인한다.

1. QR 기반 근로자 전자확인
2. 확인 내용 snapshot 저장
3. 현장 불편·개선의견 빠른 제출
4. 익명 의견 선택
5. 관리자 접수함 확인
6. 상부 보고용 주간 요약 copy 후보 확인

아래 기능은 1차 제외다.

- 손가락 서명패드
- 본인인증 연동
- HACCP 전체 문서관리
- MSDS RAG
- 소방·전기 점검 조치 추적
- 가열실 IoT 센서 API 연동
- 정식 PDF 보고서 자동생성
- 다국어 자동 번역

## 5. QR 링크 확인

Companies DB 세팅 후 아래 경로를 확인한다.

    /field/participation?company=richi

확인 항목:

- badge가 리치코리아 현장 전자확인으로 표시되는지
- title이 리치코리아 현장 전자확인·피드백으로 표시되는지
- 안내문이 QR 기반 전자확인 안내로 표시되는지
- 버튼 문구가 전자확인·의견 제출하기로 표시되는지
- 법적 효력 보장 또는 종이서명 완전 대체 표현이 없는지
- 익명 제출 선택이 의견 제출 시 정상 표시되는지
- 제출 완료 화면에서 리치코리아 문구가 표시되는지

## 6. 저장 테스트 체크리스트

실제 고객 데이터와 섞이지 않도록 테스트 입력값을 명확히 표시한다.

권장 테스트 값:

- 작성자: 테스트근로자A
- 소속 또는 작업조: 리치테스트조
- 위치/구역: 테스트구역
- 제목: 리치코리아 QR 전자확인 테스트
- 내용: 체험판 저장 흐름 확인용 테스트 기록

확인 항목:

- 제출이 오류 없이 완료되는지
- 관리자 접수함에서 companyCode=richi 기준으로 확인 가능한지
- 익명 제출 시 작성자 정보가 노출되지 않는지
- 비익명 제출 시 최소 확인정보가 기록되는지
- 사진 첨부 시 용량 조정 및 저장 흐름이 깨지지 않는지
- Supabase shadow-write가 켜져 있다면 field_participation_submissions row가 생성되는지
- Notion 현장참여 DB 저장이 필요한 경우 해당 DB에 기록되는지

## 7. 관리자 확인 체크리스트

관리자 화면에서는 아래 기준으로 확인한다.

- 리치코리아 제출 건만 분리해서 볼 수 있는지
- 공유확인과 현장 의견이 구분되는지
- 불편사항·개선의견·위생안전 확인·기타 유형이 운영상 충분한지
- 조치 메모 또는 처리상태가 필요한지
- 주간 요약에 들어갈 문구가 과하지 않은지

## 8. 요청자료 체크리스트

리치코리아에 요청할 자료는 아래와 같다.

1. 근로자 서명부 또는 교육 참석 확인 양식
2. 작업 전 위생·안전 체크리스트
3. HACCP 점검표
4. 일일 작업일지 또는 생산·위생 확인 양식
5. 현장 불편·개선 요청 양식
6. 관리자 조치 기록 양식
7. 상부 보고자료 샘플
8. 월별 점검·교육·개선 보고자료
9. 근로자가 휴대폰을 사용할 수 있는 구역과 시간
10. 가열실 온습도계 또는 도입 예정 센서 자료
11. 외국인 근로자 비율과 필요한 언어 후보

개인정보, 직원명, 전화번호, 내부 민감정보는 가리고 받아도 된다.

## 9. 다국어 후속 기준

다국어는 1차 체험판 범위에 넣지 않는다.

후속 후보로 아래 화면부터 검토한다.

1. QR 첫 화면 안내문
2. 전자확인 버튼과 동의 문구
3. 현장 불편·개선의견 제출 항목
4. 개인정보 안내 요약
5. 제출 완료 화면

다국어는 현장 이해도 개선 기능이다. 자동 번역 정확성 보장이나 법적 효력 보장 표현은 사용하지 않는다.

## 10. 금지 사항

아래 내용은 문서, 화면, 제안서, 영업 문구에 사용하지 않는다.

- 법적 면책
- 처벌 방지
- 무재해 보장
- HACCP 인증 보장
- 종이서명 완전 대체 보장
- 전자확인 법적 효력 보장
- 위험성평가 대행
- 안전관리대행
- AI가 조치 완료 또는 법적 판단을 확정한다는 표현

## 11. 보안 기준

아래 값은 문서, 채팅, GitHub, PR 본문에 기록하지 않는다.

- 토큰 실제 값
- API Key 실제 값
- service role 실제 값
- Owner Token 실제 값
- 환경변수 실제 값
- 고객 개인정보 원문
- 실제 직원명, 전화번호, 민감한 내부 자료 원문

## 12. 완료 기준

이 체크리스트 기준 완료 조건은 아래와 같다.

- Companies DB에 companyCode=richi 기준 등록 여부 확인
- active=true 확인
- 필요한 DB 연결 필드 확인
- /field/participation?company=richi 화면 진입 확인
- 리치코리아 전용 copy 표시 확인
- 테스트 제출 1건 이상 저장 확인
- 관리자 접수함 확인
- 금지 표현 미사용
- 토큰/API Key/Owner Token/환경변수 실제 값 미기록
