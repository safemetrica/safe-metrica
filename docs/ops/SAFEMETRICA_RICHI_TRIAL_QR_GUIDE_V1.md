# SafeMetrica Richi 1-Week Trial QR Guide v1

## 1. 목적

이 문서는 리치코리아 1주일 체험판에서 근로자 QR 전자확인·현장 피드백 흐름을 안내하기 위한 운영가이드다.

체험판 목표는 정식 HACCP 문서관리 시스템 구축이 아니라, 식품가공 현장에서 근로자가 10~30초 안에 확인하고 의견을 남길 수 있는지 검증하는 것이다.

## 2. 체험판 포지션

고객용 표현:

    리치코리아 현장 전자확인·피드백 운영앱

근로자용 표현:

    QR 기반 전자확인 기록
    현장 불편·개선의견 제출

관리자용 표현:

    근로자 확인과 현장 피드백을 접수하고 주간 요약 후보로 정리하는 체험판

사용하지 않을 표현:

- 전자서명 법적 효력 보장
- 종이서명 완전 대체
- HACCP 인증 보장
- 사고 방지 보장
- 처벌 방지
- 법적 면책

## 3. 체험판 QR 링크

기본 경로:

    /field/participation?company=richi

운영 전제:

- Companies DB에 companyCode=richi가 등록되어 있어야 한다.
- active=true 상태여야 한다.
- 현장참여 저장 DB 연결 필드가 확인되어야 한다.
- 실제 DB ID, 토큰, API Key, 환경변수 값은 문서에 기록하지 않는다.

## 4. 근로자 안내문

현장 게시 또는 구두 안내용 문구:

    오늘 작업 전 확인할 내용을 QR로 확인하고, 불편한 점이나 개선의견이 있으면 짧게 남겨 주세요.
    의견이 없으면 확인만 남길 수 있습니다.
    제출 내용은 관리자 확인과 주간 요약용 운영기록으로 활용됩니다.

짧은 버전:

    QR 접속 → 확인 → 의견 있으면 제출 → 관리자가 확인

## 5. 근로자 입력 유형

리치코리아 체험판 의견유형은 아래 4개로 둔다.

1. 불편사항
2. 개선의견
3. 위생·안전 확인
4. 기타

운영 판단:

- 위험제보라는 표현은 식품가공 체험판 초기에 부담이 클 수 있다.
- 리치코리아는 불편사항과 개선의견 중심으로 입력 저항을 낮춘다.
- 위생·안전 확인은 HACCP 현장 특성을 반영하되 인증 보장 표현으로 쓰지 않는다.

## 6. 관리자 확인 기준

관리자는 제출 건을 아래 기준으로 본다.

- 근로자가 QR에 정상 접근했는지
- 10~30초 안에 입력 가능한지
- 불편사항과 개선의견이 실제로 접수되는지
- 익명 제출이 필요한 상황이 있는지
- 사진 첨부가 필요한지
- 관리자 피드백 문구가 필요한지
- 주간 요약에 넣을 만한 항목이 있는지

## 7. 테스트 제출 기준

체험판 전 내부 테스트는 실제 고객 데이터와 섞이지 않게 한다.

권장 테스트 값:

- 작성자: 테스트근로자A
- 소속 또는 작업조: 리치테스트조
- 위치/구역: 테스트구역
- 제목: 리치코리아 QR 전자확인 테스트
- 내용: 체험판 저장 흐름 확인용 테스트 기록

테스트 완료 후 확인할 항목:

- 제출 완료 화면 표시
- 관리자 접수함 표시
- companyCode=richi 기준 분리
- 익명 제출 동작
- 비익명 제출 시 최소 확인정보 저장
- 사진 첨부 동작
- Supabase shadow-write 사용 시 field_participation_submissions row 생성 여부
- Notion 현장참여 DB 저장이 필요한 경우 해당 DB 기록 여부

## 8. 체험판 제외범위

아래는 1주일 체험판 범위에서 제외한다.

- 손가락 서명패드
- 본인인증 연동
- HACCP 전체 문서관리
- MSDS RAG
- 소방·전기 점검 조치 추적
- 가열실 IoT 센서 API 연동
- 정식 PDF 보고서 자동생성
- 다국어 자동 번역

## 9. 다국어 후속 기준

다국어는 1차 체험판 이후 검토한다.

후속 후보:

- QR 첫 화면 안내문
- 전자확인 버튼과 확인 문구
- 불편사항·개선의견 제출 항목
- 개인정보 안내 요약
- 제출 완료 화면

다국어는 현장 이해도 개선 기능으로 표현한다. 자동 번역 정확성 보장 또는 법적 효력 보장 표현은 사용하지 않는다.

## 10. 고객에게 요청할 자료

리치코리아에 요청할 자료:

1. 근로자 서명부 또는 교육 참석 확인 양식
2. 작업 전 위생·안전 체크리스트
3. HACCP 점검표
4. 일일 작업일지 또는 생산·위생 확인 양식
5. 현장 불편·개선 요청 양식
6. 관리자 조치 기록 양식
7. 상부 보고자료 샘플
8. 월별 점검·교육·개선 보고자료
9. 근로자가 휴대폰을 사용할 수 있는 구역과 시간
10. 가열실 온습도계 또는 도입 예정 센서 자료
11. 외국인 근로자 비율과 필요한 언어 후보

개인정보, 직원명, 전화번호, 내부 민감정보는 가리고 받아도 된다.

## 11. 주간 요약 후보

체험판 종료 후 아래 항목을 요약한다.

- 총 QR 접속/제출 건수
- 확인만 제출한 건수
- 불편사항 건수
- 개선의견 건수
- 위생·안전 확인 건수
- 익명 제출 건수
- 사진 첨부 건수
- 관리자 확인 필요 항목
- 정식 도입 시 보완할 UX 항목

## 12. 금지 표현

아래 표현은 사용하지 않는다.

- 법적 면책
- 처벌 방지
- 무재해 보장
- HACCP 인증 보장
- 종이서명 완전 대체 보장
- 전자확인 법적 효력 보장
- 위험성평가 대행
- 안전관리대행
- AI가 법적 판단 또는 조치완료를 확정한다는 표현

## 13. 보안 기준

아래 값은 문서, 채팅, GitHub, PR 본문에 기록하지 않는다.

- 토큰 실제 값
- API Key 실제 값
- service role 실제 값
- Owner Token 실제 값
- 환경변수 실제 값
- 고객 개인정보 원문
- 실제 직원명, 전화번호, 민감한 내부 자료 원문

## 14. 완료 기준

이 가이드 기준 완료 조건은 아래와 같다.

- 체험판 QR 링크 확정
- 근로자 안내문 확정
- 리치코리아 의견유형 4개 확인
- 내부 테스트 제출 1건 이상 확인
- 관리자 접수함 확인
- 고객 요청자료 목록 전달
- 금지 표현 미사용
- 토큰/API Key/Owner Token/환경변수 실제 값 미기록
