# SafeMetrica Partner Demo Truth Map

## 1. 한 줄 정의

`Partner Demo는 실제 고객 데이터 없이 SafeMetrica의 핵심 운영 흐름을 체험하는 샘플 모드입니다.`

## 2. 체험판 원칙

| 항목 | 기준 |
| --- | --- |
| 목적 | 파트너 교육·영업 시연 |
| 데이터 | 샘플 데이터 |
| 저장 | 브라우저 localStorage 샘플 상태 |
| 실제 DB 저장 | 없음 |
| 실제 고객 데이터 | 사용 금지 |
| 정식앱과의 관계 | 정식앱 핵심 흐름의 샘플 모드 |
| 과장 기능 | 금지 |

## 3. 정식앱과 동일해야 하는 것

| 항목 | 기준 |
| --- | --- |
| 역할 구조 | 근로자 / 현장관리자 / 대표 |
| 운영 흐름 | TBM → 공유확인 → 위험제보 → 조치 → 현장비서 → 대표보고 |
| 표현 | 운영기록, 공유확인, 조치이력, 월간보고서 |
| 완료 피드백 | 저장/확인 완료 상태 표시 |
| 결과 흐름 | 월간보고서와 위험성평가표 샘플로 연결 |

## 4. 체험판에서 다른 것

| 항목 | 체험판 | 정식앱 |
| --- | --- | --- |
| 데이터 | 샘플 데이터 | 고객사 실제 데이터 |
| 저장 | localStorage | 고객사 전용 DB/운영 저장소 |
| 사진 | 샘플 사진/업로드 시뮬레이션 | 실제 사진첨부 및 저장 |
| 음성 | 녹음 흐름 시뮬레이션 가능 | 실제 음성작성 기능 |
| 보고서 | 샘플 보고서 | 실제 월간 운영기록 기반 보고서 |
| 권한 | 자유 체험 | 역할·고객사별 제한 |

## 5. 체험판에서 절대 하면 안 되는 것

* 실제 고객명 사용 금지
* 실제 고객 사진 사용 금지
* 실제 고객 DB 연결 금지
* Owner Console 노출 금지
* Notion/Supabase/API 직접 호출 금지
* token/key/env 노출 금지
* 법적 면책, 처벌 방지, 완벽 대응, 자동 확정 표현 금지
* 정식앱에 없는 기능을 있는 것처럼 표현 금지

## 6. 체험판 필수 안내문

`이 체험판은 실제 고객 데이터 없이 구성된 샘플 운영 흐름입니다.`

`정식 도입 시 고객사 전용 환경에서 실제 TBM, 공유확인, 조치이력, 월간보고서가 저장됩니다.`

`체험판에서는 브라우저 안에서만 상태가 반영되며 실제 고객 DB에는 저장되지 않습니다.`

## 7. 체험판 최종 구조

1. Partner Demo 홈
2. 근로자 체험
   * TBM 확인
   * 공유확인
   * 위험제보 제출
3. 현장관리자 체험
   * 참여현황 확인
   * 위험제보 접수
   * 조치사진 샘플 저장
   * 현장비서 요약
4. 대표 체험
   * 운영현황 대시보드
   * 위험성평가표 샘플
   * 월간보고서 샘플

## 8. 다음 단계

* worker / manager / ceo 상세 화면을 Lovable 스타일 DemoLayout 기준으로 재구성
* 단, 기능 약속은 현재 정식 세메앱에서 제공 가능한 범위 안에서만 표현
* 체험판과 정식앱 차이는 `샘플 저장`과 `실제 저장`의 차이로만 유지
