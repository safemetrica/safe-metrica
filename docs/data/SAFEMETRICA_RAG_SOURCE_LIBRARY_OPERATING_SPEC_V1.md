# SafeMetrica RAG Source Library Operating Spec v1

## 1. 목적

SafeMetrica 지식허브와 Notion RAG 자료 라이브러리에 업로드되는 정부·공공기관 기준 자료의 운영 원칙을 정의한다.

RAG 자료는 고객 안내, 안전보건관리체계 기준 정리, 위험성평가·TBM·현장참여·대표 확인 흐름의 근거 관리, 사고사례 기반 TRIGGER DB 고도화, 폭염·한파 등 시즌 SOP 고도화에 사용한다.

AI는 법적 판단자가 아니라 후보 제안자이며, 최종 판단과 책임은 관리자와 사업주에게 있다.

## 2. Notion 위치

📘 운영 기준 하위에 📚 RAG 자료 라이브러리 페이지를 둔다.

SafeMetrica 루트에는 자료를 계속 쌓지 않고, 운영 기준 하위의 별도 RAG 자료 라이브러리로 관리한다.

## 4. RAG 운영 단계

| 단계 | 명칭 | 설명 |
| --- | --- | --- |
| L0 | Raw Attachment | 원문 PDF 보관 |
| L1 | Source Index | 제목, 출처, 발간일, 적용범위 정리 |
| L2 | Core Extract | 핵심 기준과 Q&A를 사람이 검수해 요약 |
| L3 | SafeMetrica Mapping | 앱 기능, DB, 보고서와 연결 |
| L4 | Structured DB | TRIGGER DB, 위험요인 MASTER, SOP 모듈화 |

이번 v1 범위는 L1~L2 중심이다. 중대재해 사이렌은 L1까지만 처리한다.

## 6. v1에서 하지 않는 것

- 사이렌 전체 OCR 전수 파싱
- 법적 판단 자동화
- 고객에게 면책 또는 처벌 방지 표현 제공
- 원문 전체를 앱 답변에 직접 노출
- 고객 민감정보와 정부자료 RAG 혼합
- Owner/Admin 링크 또는 내부 DB 구조 노출
- API Key, 토큰, 환경변수 노출

## 7. 금지 표현

법적 면책 보장, 처벌 방지, 과태료 방지, 중대재해 면책, 무재해 보장, KOSHA 인정 보장, 위험성평가 대행, 안전관리대행, AI가 법적 조치를 확정, AI가 조치 완료를 확정, 대표 책임 제거, 사고 시 문제없음.

## 8. 권장 표현

정부 기준 자료를 참고한 운영기록 후보입니다.

현장 입력, 관리자 확인, 대표 확인 흐름을 기록으로 남깁니다.

AI는 후보 제안자이며 최종 판단과 책임은 관리자와 사업주에게 있습니다.

안전보건 관리·조치·확인 과정을 월간 운영자료로 정리합니다.
