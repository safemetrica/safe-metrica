# SafeMetrica Mobile TBM / Field Assistant Compact UX Spec v1

기준일: 2026-06-09
문서 위치: `docs/data/SAFEMETRICA_MOBILE_TBM_FIELD_ASSISTANT_COMPACT_UX_SPEC_V1.md`
작업명: `docs: add mobile tbm field assistant compact ux spec v1`

## 1. 목적

이 문서는 SafeMetrica 정식앱의 모바일 TBM 화면과 현장비서 화면을 현장관리자가 실제 휴대폰에서 빠르게 확인할 수 있도록 압축 표시 기준을 정리하기 위한 문서다.

PC 화면에서는 상세 정보가 어느 정도 표시되어도 무리가 없지만, 모바일 화면에서는 TBM 원문, STT 원문, AI 분석 내용, 위험요인, 사진증빙, 교육시간 후보가 한 화면에 길게 노출될 경우 현장관리자의 피로도가 커진다.

따라서 v1에서는 모바일 화면을 다음 원칙으로 정리한다.

```text
모바일은 요약 먼저,
상세는 접기,
원문은 숨김,
사진은 별도 카드,
교육시간은 후보 배지,
기상위험은 위험 단계일 때만 상단 노출.
```

## 2. 현재 문제

현재 모바일 TBM 화면과 현장비서 화면에서 확인된 문제는 다음과 같다.

1. STT 원문과 AI 분석 문장이 길게 표시되어 글자 중압감이 크다.
2. 현장관리자가 작업 전 빠르게 봐야 할 정보와 상세 검토 정보가 섞여 있다.
3. 위험요인이 문장형으로 길게 노출되어 핵심 위험을 즉시 파악하기 어렵다.
4. 사진증빙 정보가 TBM 본문과 섞여 화면 스크롤이 길어진다.
5. 교육시간 후보, TBM 인정 후보, 증빙 후보가 확정값처럼 보일 위험이 있다.
6. 기상위험 카드가 평상시에도 과하게 상단 노출되면 피로도가 생긴다.
7. 대표, 현장관리자, 근로자가 봐야 할 정보의 우선순위가 다르다.

## 3. 기본 원칙

### 3-1. 모바일 우선 정보

모바일 화면은 아래 5가지를 먼저 보여준다.

```text
1. 오늘 작업명
2. 핵심 위험요인 3~5개
3. 오늘 주의사항 3줄 요약
4. 조치 필요 여부
5. 사진증빙 상태
```

### 3-2. 상세 정보는 접기

아래 항목은 모바일에서 기본 접힘 처리한다.

```text
STT 원문
AI 분석 전체문
상세 위험요인 설명
긴 교육 문구
상세 사진 파일 목록
원본 기록 메타데이터
```

### 3-3. AI 표현 기준

AI는 법적 판단자나 조치 확정자가 아니다.

모바일 화면에서도 아래 표현은 사용하지 않는다.

```text
교육 이수 완료
법정교육 인정 완료
위험성평가 이행 완료
조치 완료 확정
법적 증빙 완료
과태료 방지
중대재해 면책
무재해 보장
KOSHA 인정 보장
```

권장 표현은 다음과 같다.

```text
교육시간 인정 후보
TBM 기록 후보
증빙 보완 필요
관리자 확인 필요
위험성평가 공유 확인 기록
현장 확인 기록
```

## 4. PC / 모바일 표시 기준

| 구분 | PC | 모바일 |
| --- | --- | --- |
| TBM 제목 | 전체 표시 | 1~2줄 표시 |
| 작업내용 | 상세 표시 가능 | 3줄 요약 우선 |
| 위험요인 | 문장 + 태그 | 태그 3~5개 우선 |
| STT 원문 | 펼침 가능 | 기본 숨김 |
| AI 분석 | 카드 표시 | 요약만 표시 |
| 사진증빙 | 상세 목록 가능 | 별도 카드 |
| 교육시간 후보 | 설명 포함 가능 | 배지 표시 |
| 기상위험 | 하단/중단 카드 | Level 2 이상 상단 이동 |
| 조치상태 | 상세 표시 | 배지 + CTA |

## 5. 모바일 TBM 카드 구조

모바일 TBM 카드는 다음 순서로 구성한다.

```text
[상단]
- 작업명
- 날짜 / 시작시간
- 작업유형 배지

[핵심 요약]
- 오늘 작업 1줄
- 핵심 위험요인 태그 3~5개
- 오늘 주의사항 3줄

[상태 배지]
- 특이사항 없음 / 있음
- 조치 필요 / 조치완료 / 확인 필요
- 교육시간 인정 후보
- 사진증빙 있음 / 보완 필요

[CTA]
- 상세보기
- 원문보기
- 사진증빙 보기
```

## 6. TBM 요약 규칙

### 6-1. 3줄 요약 기준

모바일 TBM 요약은 다음 3줄을 기본으로 한다.

```text
1줄: 오늘 작업 내용
2줄: 가장 중요한 위험요인
3줄: 작업 전 확인 또는 주의사항
```

예시:

```text
오늘 작업: 생활폐기물 수거 및 차량 이동 작업
핵심 위험: 후진 차량, 적재함 협착, 미끄럼
확인사항: 유도자 확인, 적재함 주변 접근 금지, 보호구 착용
```

### 6-2. 위험요인 태그 기준

위험요인은 최대 5개까지만 우선 노출한다.

```text
후진
협착
미끄럼
찔림·베임
온열질환
```

5개를 초과하는 경우 “+N개”로 표시하고 상세보기에서 확인한다.

### 6-3. 특이사항 표시 기준

특이사항은 체크박스 true 또는 명확한 사고·아차사고·이상상황이 있을 때만 강조한다.

일반적인 교육 문구, 보호구 착용 안내, 서행 운전, 정리정돈, 스트레칭 안내는 특이사항 강조 대상이 아니다.

## 7. 숨김 처리 대상

모바일 기본 화면에서 숨길 항목은 다음과 같다.

```text
STT 원문 전체
AI 분석 전문
원본 음성 변환 텍스트
긴 오늘주의사항 원문
내부 ID
Notion URL
Supabase UUID
raw payload
환경변수명
API 경로
Owner/Admin 전용 링크
```

단, 관리자가 필요할 경우 “원문보기” 또는 “상세보기” 버튼으로 확인할 수 있게 한다.

## 8. 상세보기 구조

모바일 상세보기는 한 번에 모두 펼치지 않고 섹션별로 나눈다.

```text
1. 작업 내용 상세
2. 위험요인 상세
3. 조치상태
4. 사진증빙
5. 교육/TBM 인정 후보
6. STT 원문
```

STT 원문은 가장 아래에 배치하고 기본 접힘 처리한다.

## 9. 현장비서 모바일 구조

현장비서 모바일 화면은 “오늘 먼저 볼 것” 중심으로 정리한다.

### 9-1. 기본 구조

```text
[오늘 먼저 확인]
- TBM 미작성
- 조치 필요
- 현장참여 검토 필요
- 사진증빙 보완 필요
- 기상위험 Level 2 이상

[오늘 작업 요약]
- 작업명
- 위험요인 태그
- 확인 버튼

[나중에 확인]
- 뉴스
- KOSHA 참고자료
- 긴 안내문
```

### 9-2. 현장관리자 CTA

현장관리자에게 필요한 CTA는 다음으로 제한한다.

```text
TBM 작성하기
제보 확인하기
조치 기록하기
사진증빙 보기
위험성평가 확인하기
```

## 10. 기상 위험 카드 위치 기준

기상 위험 카드는 위험 단계에 따라 위치를 조정한다.

| Level | 상태 | 모바일 위치 |
| --- | --- | --- |
| Level 0 | 일반 | 하단 요약 카드 |
| Level 1 | 참고 | 하단 또는 중단 |
| Level 2 | 주의 | 오늘 할 일 아래 |
| Level 3 | 경계 | 오늘 할 일 위 긴급 카드 |
| Level 4 | 심각 | 최상단 긴급 카드 |

## 11. 기상 위험 카드 문구 기준

기상 위험 문구는 확정 명령이 아니라 작업 전 확인 안내로 표시한다.

권장 문구:

```text
오늘 기상 조건상 작업 전 확인이 필요합니다.
현장관리자는 작업 전 휴식, 보호구, 작업시간, 작업자 이상증상 여부를 확인하세요.
```

폭염 예시:

```text
폭염 주의 조건입니다.
작업 전 물·그늘·휴식 준비상태와 근로자 자각증상을 확인하세요.
```

강풍 예시:

```text
강풍 위험 조건입니다.
고소작업, 적재물, 외부 작업구역의 작업 전 확인이 필요합니다.
```

금지 표현:

```text
작업중지 명령 자동 발동
법정 작업중지 기준 충족
과태료 방지 조치 완료
안전조치 완료 확정
```

## 12. 교육/TBM 인정 후보 배지

TBM 기록이 교육시간 인정 후보로 검토될 수 있는 경우 모바일에서는 배지로 표시한다.

표시 예시:

```text
교육시간 인정 후보
TBM 기록 후보
사진증빙 있음
관리자 확인 필요
```

주의:

```text
교육시간 인정 여부는 앱이 확정하지 않는다.
사업장 조건, 교육 내용, 대상자, 시간, 증빙, 관련 기준 확인 후 판단한다.
```

## 13. 위험성평가 공유·참여 QR 상품 연결

모바일 압축 UX는 향후 단품 상품인 SafeMetrica Risk Share Link와 연결될 수 있다.

상품 후보:

```text
SafeMetrica Risk Share Link
위험성평가 공유·참여 증빙 QR팩
근로자 참여 증빙 QR팩
```

기본 흐름:

```text
근로자 QR 접속
→ 오늘 위험요인 확인
→ 위험성평가 공유 확인
→ 안전조치 주지 확인
→ 의견 없음 또는 위험제보 제출
→ 관리자 접수함 반영
→ 월간보고서/고객 Export 반영
```

가격 후보는 별도 상품 Spec에서 검토한다.

```text
월 2,000원/인 후보
월 최소요금 20만 원 후보
초기 세팅비 30만~50만 원 후보
```

단, 가격은 확정값이 아니라 영업 검토 후보로 둔다.

## 14. v1에서 하지 않는 것

v1에서는 아래 기능을 구현하지 않는다.

```text
실제 푸시 알림
실제 사이렌
강제 알림음
반복 진동
작업중지 자동 명령
법정교육 인정 자동 확정
교육시간 자동 확정
AI 조치완료 자동 확정
Vision AI 사진 자동 확정
위치 기반 강제 인증
```

v1은 화면 구조와 표시 기준을 잠그는 문서다.

## 15. 후속 작업

1. 모바일 TBM 3줄 요약 구현
2. STT 원문 기본 숨김 구현
3. 위험요인 태그 3~5개 제한 표시
4. 사진증빙 카드 분리
5. 교육시간 후보 배지 구현
6. 현장비서 모바일 요약 카드 구현
7. 기상 위험 Level 2 이상 상단 이동 구현
8. Risk Share Link 상품 Spec v1 작성
9. 위험성평가 공유·참여 QR UX Spec v1 작성
10. Supabase Evidence ZIP Export Spec v2 작성

## 16. 최종 정리

모바일 TBM과 현장비서는 “많이 보여주는 화면”이 아니라 “작업 전 빠르게 판단하게 돕는 화면”이어야 한다.

SafeMetrica의 모바일 UX 방향은 다음으로 고정한다.

```text
3줄 요약
태그 중심
원문 숨김
상세 분리
사진 카드 분리
교육·증빙은 후보 표시
위험 단계일 때만 긴급 노출
```
