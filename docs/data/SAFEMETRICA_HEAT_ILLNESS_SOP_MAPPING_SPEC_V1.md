# SafeMetrica Heat Illness SOP Mapping Spec v1

## 1. 목적

이 문서는 고용노동부 폭염 대비 온열질환 예방 사업장 대응지침을 SafeMetrica 앱 기능에 어떻게 매핑할지 정의한다.

SafeMetrica는 폭염 기준을 법적 판단으로 확정하지 않는다. 앱은 현장관리자와 대표가 참고할 수 있는 운영기록 후보와 확인 항목을 제안한다.

AI는 후보 제안자이며, 최종 판단과 책임은 관리자와 사업주에게 있다.

## 2. 기준 자료

- 폭염 대비 온열질환 예방을 위한 사업장 대응지침
- 발간: 고용노동부, 2025.7
- 적용 범위: 폭염작업, 온열질환 예방, 휴식, 체감온도 기록, 응급조치, 사업장 예방체계

| --- | --- | --- |
| 체감온도 31도 이상 | 폭염작업 주의 | 2시간 이상 작업 시 보건조치 검토 |
| 체감온도 33도 이상 | 휴식 필요 | 매 2시간 이내 20분 이상 휴식 원칙 |

| 체감온도 38도 이상 | 매우 위험 권고 | 긴급조치 외 옥외작업 중지 검토 |
| 온열질환 의심 | 즉시 조치 | 119 신고 및 응급조치 후보 |

## 5. 날씨 카드 표시 기준

날씨 카드는 판단을 확정하지 않고 운영 신호로 표시한다.

| 체감온도 | 표시 문구 후보 |
| --- | --- |
| 31도 미만 | 정상 작업 |
| 31도 이상 | 폭염작업 가능성 확인 |
| 33도 이상 | 휴식 부여 확인 필요 |
| 35도 이상 | 고위험 폭염 대응 필요 |
| 38도 이상 | 매우 위험, 긴급작업 외 옥외작업 제한 검토 |

## 6. TBM 문구 후보

### 31도 이상

오늘은 체감온도가 높아 폭염작업 가능성이 있습니다. 작업 전 물, 휴식장소, 통풍 또는 그늘 조치 여부를 확인해 주세요.

### 33도 이상

체감온도 33도 이상 작업장소에서는 휴식 부여 여부 확인이 필요합니다. 현장 여건에 따라 2시간 이내 20분 이상 휴식 원칙을 검토해 주세요.

### 35도 이상

고위험 폭염 상황입니다. 무더위 시간대 옥외작업 최소화, 매시간 휴식, 온열질환 민감군 상태 확인을 검토해 주세요.

### 38도 이상

매우 위험한 폭염 상황입니다. 긴급조치 외 옥외작업 제한 여부와 작업 중지·대체작업 가능성을 우선 검토해 주세요.

## 7. 근로자 자각증상 체크 후보

근로자 확인 화면 또는 현장참여 화면에서 아래 증상 체크 후보를 제공할 수 있다.

- 두통
- 어지러움
- 메스꺼움
- 근육경련
- 지나친 땀
- 갑작스러운 피로감
- 의식 저하
- 구역질

2개 이상 증상이 있거나 상태 판단이 어려운 경우 관리자에게 즉시 알리고 필요 시 119 신고 및 응급조치를 검토한다.

## 8. 사진증빙 분류 후보

| 증빙유형 | 예시 |
| --- | --- |
| 물/음료수 | 생수, 얼음물, 이온음료 비치 |
| 그늘/휴식 | 그늘막, 휴게시설, 냉방 차량 |
| 온습도 측정 | 온습도계, 체감온도 확인 화면 |
| 통풍/냉방 | 선풍기, 냉풍기, 이동식 에어컨 |
| 보냉장구 | 아이스조끼, 냉각장구 |
| 응급대응 | 응급키트, 119 신고 안내문 |

## 9. 월간보고서 반영 후보

월간보고서에는 아래 항목을 요약할 수 있다.

- 폭염 주의일 수
- 체감온도 기준 초과일
- TBM 폭염 안내 반영 횟수
- 물·그늘·휴식 증빙 수
- 온열질환 자각증상 제보 수
- 미조치 또는 확인 필요 항목
- 다음 달 개선 후보

## 10. v1에서 하지 않는 것

- 법적 이행 완료 자동 판정
- 휴식 부여 의무 충족 여부 확정
- 온열질환 발생 책임 판단
- 고객에게 면책 또는 처벌 방지 표현 제공
- 기상청·고용노동부 원문을 고객 화면에 장문 노출
- 민감 건강정보의 장기 보관 구조 확정

## 11. 금지 표현

- 법적 면책 보장
- 처벌 방지
- 과태료 방지
- 온열질환 사고 방지 보장
- 무재해 보장
- AI가 조치 완료를 확정
- 대표 책임 제거
- 사고 시 문제없음

## 12. 권장 표현

- 정부 기준 자료를 참고한 폭염 대응 운영기록 후보입니다.
- 현장 여건에 따라 관리자 확인 후 조치하세요.
- AI는 후보 제안자이며 최종 판단과 책임은 관리자와 사업주에게 있습니다.
- 실제 기록과 처리 상태는 각 메뉴에서 확인해야 합니다.
