# SafeMetrica Weather Risk Alert Escalation Spec v1

## 1. 목적

이 문서는 SafeMetrica에서 폭염, 한파, 강수, 강풍 등 기상 리스크를 어떻게 알림 단계로 전환할지 정의한다.

SafeMetrica는 기상 리스크를 법적 판단이나 작업중지 명령으로 확정하지 않는다. 앱은 현장관리자, 대표, 근로자가 참고할 수 있는 운영 신호와 확인 후보를 제공한다.

AI는 후보 제안자이며, 최종 판단과 책임은 관리자와 사업주에게 있다.

## 2. 기준 자료

| 자료 | 활용 |
| --- | --- |
| 고용노동부 폭염 대비 온열질환 예방 사업장 대응지침 | 폭염 기준, 휴식, 119 신고, 체감온도 기록 |
| 2025 안전보건교육 안내서 | 폭염·한파작업 건강장해 응급조치 교육내용, TBM 교육기록 후보 |

| SafeMetrica Safety Education Guide Mapping Spec v1 | TBM·교육기록·증빙·월간보고서 매핑 |

## 3. 알림 원칙

1. 알림은 작업중지 명령이 아니라 확인 요청이다.
2. 알림은 위험 단계별로 강도를 다르게 한다.
3. 근로자, 현장관리자, 대표의 알림 목적을 분리한다.
4. 실제 사이렌, 강제 알림음, 반복 진동은 후속 단계로 분리한다.
5. 고객 화면에는 법적 면책, 처벌 방지, 무재해 보장 표현을 사용하지 않는다.
6. 알림 발생·확인·조치 상태는 월간보고서와 Evidence Book 후보로 연결한다.

## 4. 역할별 알림 방향

| 역할 | 알림 목적 | v1 알림 방식 |
| --- | --- | --- |
| 근로자 | 작업 전 위험 인지, 자각증상 확인 | 작업 전 확인 카드, 체크리스트 |
| 현장관리자 | 즉시 확인, 현장 조치, TBM 반영 | 긴급 카드, 확인 버튼, 진동 후보 |
| 대표 | 운영상 미확인·미조치 신호 확인 | 대시보드 점등, 고위험 요약 카드 |
| Owner 내부 | 고객사별 기상 리스크 현황 파악 | 내부 운영 로그 후보 |

## 5. 기상 리스크 유형

| 유형 | 주요 리스크 | 앱 반영 후보 |
| --- | --- | --- |
| 폭염 | 온열질환, 탈수, 열사병 | 체감온도 단계, 휴식, 물, 그늘, 119 |
| 한파 | 저체온증, 동상, 미끄러짐 | 방한장비, 휴식, 결빙 확인 |
| 강수 | 미끄러짐, 감전, 시야 저하 | 노면상태, 전기작업 주의, 배수 확인 |
| 강풍 | 낙하물, 전도, 크레인·고소작업 위험 | 고소작업, 적재물, 가설물 확인 |
| 폭설 | 결빙, 붕괴, 차량사고 | 제설, 적재하중, 이동동선 확인 |
| 미세먼지 | 호흡기 건강장해 | 마스크, 실외작업 조정 후보 |

## 6. 알림 단계

| 단계 | 명칭 | 의미 | 앱 표시 |
| --- | --- | --- | --- |
| Level 0 | 정상 | 일반 확인 | 일반 날씨 카드 |
| Level 1 | 주의 | 작업 전 공유 필요 | 주의 배지 |
| Level 2 | 확인 필요 | 관리자 확인 필요 | 현장관리자 확인 카드 |
| Level 3 | 고위험 | 대표 요약 신호 필요 | 관리자 긴급 카드 + 대표 점등 |
| Level 4 | 매우 위험 | 작업 조정 검토 필요 | 긴급 카드, 대표 고위험 요약, 후속 알림 후보 |

## 11. 현장관리자 UX 후보

현장관리자는 현장에 있으므로 즉시 확인 중심으로 설계한다.

| UX | 설명 |
| --- | --- |
| 긴급 카드 | 오늘 할 일 상단 또는 날씨 카드 아래 표시 |
| 확인 버튼 | “확인했습니다” 또는 “조치 필요” |
| TBM 반영 | 오늘 TBM 문구에 자동 후보 추가 |
| 진동 후보 | PWA 알림 권한과 모바일 환경 검토 후 후속 |
| 조치 메모 | 물, 휴식, 그늘, 제설, 배수, 낙하물 제거 등 입력 |

## 12. 대표 UX 후보

대표는 반복 알림보다 운영 신호 중심으로 설계한다.

| UX | 설명 |
| --- | --- |
| 대시보드 점등 | 고위험 또는 미확인 건이 있을 때 표시 |
| 고위험 요약 카드 | 폭염·한파·강수·강풍별 요약 |
| 미확인 상태 | 현장관리자 확인 전까지 표시 |
| 월간보고서 반영 | 기상 리스크 발생일, 조치, 증빙 요약 |
| 중요 알림 후보 | Level 3 이상만 후속 검토 |

## 13. 근로자 UX 후보

근로자는 간단한 확인 중심으로 설계한다.

| UX | 설명 |
| --- | --- |
| 작업 전 확인 | 오늘 기상 리스크 확인 |
| 자각증상 체크 | 폭염·한파 건강 이상 여부 |
| 위험제보 연결 | 미끄럼, 결빙, 침수, 낙하물 제보 |
| TBM 확인 | 오늘 TBM에서 기상 리스크 공유 확인 |

## 14. 알림 기록 데이터 후보

| 필드 | 설명 |
| --- | --- |
| alert_id | 알림 후보 ID |
| company_code | 고객사 |
| site_id | 현장 |
| weather_type | 폭염/한파/강수/강풍 등 |
| alert_level | Level 0~4 |
| source_weather | 날씨 API 또는 수동 입력 |
| triggered_at | 발생일시 |
| manager_confirmed_at | 관리자 확인일시 |
| manager_action_memo | 조치 메모 |
| ceo_seen_at | 대표 확인일시 |
| monthly_report_candidate | 월간보고서 반영 후보 |
| evidence_count | 증빙 수 |

## 15. 월간보고서 매핑

월간보고서에는 아래 항목을 요약할 수 있다.

- 기상 리스크 발생일
- Level 2 이상 알림 건수
- 관리자 확인 건수
- 미확인 건수
- 조치 메모
- 증빙 수
- 대표 확인 여부
- 다음 달 보완 후보

## 16. v1에서 하지 않는 것

- 실제 사이렌 작동
- 강제 알림음
- 반복 진동
- 작업중지 명령 자동화
- 법적 이행 완료 판단
- 사고 예방 보장
- 고객에게 면책 또는 처벌 방지 표현 제공
- 민감 건강정보 장기 보관
- 외부 푸시 서비스 구현

## 17. 금지 표현

- 사고 방지 보장
- 무재해 보장
- 법적 면책 보장
- 처벌 방지
- 작업중지 자동 명령
- AI가 조치 완료를 확정
- 대표 책임 제거
- 기상 리스크 대응 완료 보장

## 18. 권장 표현

- 기상 리스크 확인이 필요한 운영 신호입니다.
- 현장 여건에 따라 관리자 확인 후 조치하세요.
- AI는 후보 제안자이며 최종 판단과 책임은 관리자와 사업주에게 있습니다.
- 실제 기록과 처리 상태는 각 메뉴에서 확인해야 합니다.

## 19. 후속 작업

1. 홈 날씨 카드 Weather Risk Level 표시
2. 현장관리자 긴급 카드 설계
3. 대표 대시보드 점등 카드 설계
4. 근로자 작업 전 기상 리스크 확인 카드 설계
5. TBM 기상 리스크 문구 후보 연결
6. 월간보고서 기상 리스크 요약 카드 설계
7. PWA 진동·푸시 알림 가능성 검토
8. 실제 사이렌·장비 연동은 Physical AI 후속 단계로 분리
